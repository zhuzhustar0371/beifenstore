## 2. 变更前分析

### 2.1 前端问题定位

- `wechat-app/pages/user/user.js`
  - “管理收货地址”“我的订单”“邀请海报”“邀请记录”“收益明细”入口未做前置登录拦截
- `wechat-app/pages/product/product.js`
  - `submitOrder()` 未校验登录状态，直接跳转地址填写页
- `wechat-app/pages/address/address.js`
  - 页面 `onShow()` 进入后直接请求 `/api/addresses`
- `wechat-app/pages/address-edit/address-edit.js`
  - `onLoad()` 进入后在下单场景直接请求 `/api/addresses/default`
- `wechat-app/utils/request.js`
  - 只对 HTTP `401` 做统一登录跳转；后端若返回 `400 + 请先登录`，只会吐司，不会统一处理

### 2.2 线上后端问题定位

- 本地代码检查发现，当前仓库已存在：
  - `backend-api/src/main/java/com/zhixi/backend/controller/AddressController.java`
  - `backend-api/src/main/java/com/zhixi/backend/service/UserAddressService.java`
  - `backend-api/src/main/java/com/zhixi/backend/mapper/UserAddressMapper.java`
- 线上接口实测：
  - `GET https://api.mashishi.com/api/auth/me` 未登录返回 `400`，消息为 `请先登录`
  - `GET https://api.mashishi.com/api/addresses` 未登录返回 `500`，消息为 `服务开小差了，请稍后重试`
  - `GET https://api.mashishi.com/api/addresses/default` 未登录返回 `500`
- 线上日志实锤：
  - `No static resource api/addresses`
  - `No static resource api/addresses/default`
- 结论：
  - 线上运行的 `app.jar` 不包含地址接口控制器，导致 `/api/addresses*` 根本没有路由，被 Spring 当作静态资源请求处理，最终由全局异常处理器包装成“服务开小差了”

## 4. 本地代码修改

### 4.1 统一鉴权失败处理

- 文件：`wechat-app/utils/request.js`
- 主要修改：
  - 新增 `isAuthMessage()` 识别 `请先登录 / 登录状态已过期 / 重新登录`
  - 新增 `showAuthToast()`，避免并发请求重复弹多次登录提示
  - 新增 `redirectToLogin()`，统一跳转登录页
  - 新增 `rejectAuthFailure()`，在鉴权失败时同时清理 `token` 与 `userInfo`
  - 扩展处理范围：
    - `200 + success:false + 请先登录`
    - `400 + 请先登录`
    - `401`

### 4.2 “我的”页入口登录拦截

- 文件：`wechat-app/pages/user/user.js`
- 主要修改：
  - 新增 `requireLogin()`
  - 对以下入口增加前置拦截：
    - `goAddress()`
    - `goOrderList()`
    - `goInvite()`
    - `goInviteRecord()`
    - `goCashback()`

### 4.3 商品详情“立即结算”登录拦截

- 文件：`wechat-app/pages/product/product.js`
- 主要修改：
  - 在 `submitOrder()` 中判断 `token`
  - 未登录时提示 `请先登录后再结算`
  - 未登录时跳转 `/pages/login/login`

### 4.4 地址列表页进入拦截

- 文件：`wechat-app/pages/address/address.js`
- 主要修改：
  - 新增 `ensureLogin()`
  - `onShow()` 中先做登录校验
  - 未登录时不再请求 `/api/addresses`

### 4.5 地址填写页进入拦截

- 文件：`wechat-app/pages/address-edit/address-edit.js`
- 主要修改：
  - 新增 `ensureLogin()`
  - `onLoad()` 首先校验登录态
  - 未登录时不再请求 `/api/addresses/default` 或 `/api/addresses/{id}`
