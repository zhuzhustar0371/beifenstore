# 原子化待操作文档

## 目标

仅通过前端样式和渲染层修复管理端表格核心编号列被挤压、长流水号竖排、发货信息列挤占其他列宽的问题。

## 修改范围

- admin-frontend/src/views/OrdersPage.vue
- admin-frontend/src/views/CashbacksPage.vue
- admin-frontend/src/styles.css
- docs/2026-04-29-admin-table-column-width-nowrap-fix/README.md

## 原子步骤

1. 订单管理：为订单号、用户编号、发货信息、操作列分配稳定 min-width。
2. 返现管理：为订单编号、微信批次、微信明细分配稳定 min-width。
3. 对核心长编号字段应用 nowrap 和省略显示，禁止逐字符换行。
4. 保留外层 overflow-x-auto，让表格总宽度超出时横向滚动。
5. 不修改批准打款、确认收款、发货、退款等业务逻辑和 API 调用。
6. 运行 npm run build 验证。
