- 用户授权：用户已明确回复“批准修复商品卡片自适应样式”。

## 二、问题分析

截图中的越界位置来自首页商品卡片：

- 页面文件：`wechat-app/pages/index/index.wxml`
- 样式文件：`wechat-app/pages/index/index.wxss`
- 加号元素：`<view class="buy-btn-icon">+</view>`

原问题原因：

1. 商品卡片使用横向 `flex` 布局。
2. 右侧 `.product-content` 没有设置 `min-width: 0`，长商品名或描述会撑开右侧内容区。
3. 底部 `.product-bottom` 没有宽度与收缩保护，价格和加号在窄屏下会被向右挤出。
4. `.buy-btn-icon` 没有 `flex-shrink: 0`，在不同机型宽度下容易被挤压或错位。
5. 之前放大的自定义底部导航栏没有小屏/大屏断点，需要补充多机型适配。

## 四、修改内容

### 1. 首页商品卡片防越界

修改文件：`wechat-app/pages/index/index.wxss`

新增/调整内容：

- `.container` 底部留白改为 `calc(220rpx + env(safe-area-inset-bottom))`，避免放大后的底部导航遮挡内容。
- `.product-card` 增加 `width: 100%`、`min-width: 0`、`align-items: stretch`、`overflow: hidden`。
- `.product-content` 增加 `min-width: 0` 和 `overflow: hidden`，允许右侧内容在窄屏内正确收缩。
- `.product-name`、`.product-desc` 增加 `display: block` 和 `max-width: 100%`。
- `.product-bottom` 增加 `width: 100%`、`min-width: 0`、`gap: 16rpx`。
- 新增 `.price-box`，设置 `flex: 1`、`min-width: 0`、`overflow: hidden`，让价格区域优先收缩。
- `.price-value` 增加省略保护，避免价格文本异常撑开布局。
- `.buy-btn-icon` 增加 `flex-shrink: 0`，保证加号按钮不被压缩、不被挤出。

### 2. 首页小屏适配

修改文件：`wechat-app/pages/index/index.wxss`

新增断点：

```css
@media screen and (max-width: 360px)
```

小屏调整：

- 缩小页面底部留白。
- 缩小顶部 banner 内边距和标题字号。
- 缩小页面左右边距。
- 商品卡片 padding 从常规 `24rpx` 降到 `18rpx`。
- 商品图从常规 `200rpx` 降到 `168rpx`。
- 商品内容左边距从 `24rpx` 降到 `18rpx`。
- 商品名、描述、价格、加号按钮按比例缩小。

### 3. 首页大屏适配

修改文件：`wechat-app/pages/index/index.wxss`

新增断点：

```css
@media screen and (min-width: 430px)
```

大屏调整：

- 页面左右边距加大。
- 商品卡片 padding 加大。
- 商品图从 `200rpx` 放大到 `220rpx`。
- 商品内容左边距加大。
- 商品名、描述、加号按钮略微放大。

### 4. 自定义底部导航多机型适配

修改文件：`wechat-app/custom-tab-bar/index.wxss`

新增小屏断点：

- `.liquid-tabbar-shell` 左右边距缩小，底部距离降低。
- 导航栏高度从常规 `128rpx` 缩到 `116rpx`。
- 圆圈从常规 `72rpx` 缩到 `64rpx`。
- 图标从常规 `38rpx` 缩到 `34rpx`。
- 文字从常规 `26rpx` 缩到 `24rpx`。

新增大屏断点：

- `.liquid-tabbar-shell` 左右边距加大。
- 导航栏高度放大到 `136rpx`。
- 圆圈放大到 `76rpx`。
- 图标放大到 `40rpx`。
- 文字放大到 `27rpx`。
