# 2026-04-20 页面层级与分组优化记录

- 变更文件：
  - `zhixi-website/frontend/src/views/HomePage.vue`

  - `zhixi-website/frontend/src/views/UserCenterPage.vue`

## 本阶段目标

- 用更清晰的留白和分组去划分页面元素。
- 强化“页面头部 -> 摘要信息 -> 明细内容”的视觉层级。
- 去掉页面中的临时样式和散落结构，让相似功能采用相似外观。

## 首页改动

### 1. Hero 改成更清晰的双栏结构

- 左侧聚焦价值主张、说明、行动按钮和三条关键承诺。
- 右侧使用单一产品信息卡，不再依赖杂乱装饰，强化重点商品信息。
- “官方商城 / 全国包邮 / 规则透明”用一致的小卡分组，信息扫描效率更高。

### 2. 分区标题全部改成统一结构

- 每个 section 都加入小号眉标题、主标题和说明文案。
- 相比原来多个区块都类似“标题 + 居中文案”的形式，现在层级更明确，也更偏左对齐。

### 3. 商品卡和返现卡统一语言

- 商品卡增加上部标签区，强调“支持返现 / 家庭洗护”。
- 返现卡改为“标题 + 说明 + 规则列表 + 跳转链接”结构，和商品卡同样采用清晰卡片分组。
- 产品兜底文案从全大写英文改成中文“官方精选”，符合“尽量不要过多使用大写字母”的要求。

### 4. 流程区改为同构步骤卡

- 四个步骤都使用统一卡片、同尺寸编号块、同样的标题和说明层级。
- 页面阅读顺序更自然，不再只是简单把流程文案堆在一起。

## 规则页改动

### 1. 新增规则摘要区

- 在详细表格前先给出四个摘要卡：最高返现、邀请结算门槛、触发方式、数据一致性。
- 这样用户可以先抓住重点，再继续阅读细则。

### 2. 规则表格前补充说明文案

- “个人下单返现”和“邀请返现”都增加了场景说明。
- 让表格不再是单纯数字表，而是有上下文、有解释的内容模块。

### 3. 温馨提示改成结构化说明

- 从单段大段文本改成清单式表达。
- 避免用户需要在一大段里找重点，也更符合“不要只用颜色表达信息”的原则。

## 用户中心改动

### 1. 账户概览更聚焦

- 把头像、昵称、手机号、登录状态、邀请码和退出操作放进同一个信息卡里。
- 邀请码改成独立信息块，和临时内联样式彻底脱离。

### 2. 统计区语义更完整

- 订单数、邀请人数、返现累计都增加辅助说明。
- 用户不必猜每个数字的含义，信息理解更直接。

### 3. 三个记录区统一成同构卡片

- “我的订单 / 邀请记录 / 返现记录”都使用统一的卡头、描述、副标题和列表结构。
- 看起来相似的区域现在功能也相似，符合你的要求。

### 4. 未登录态做成明确空状态

- 原来是临时拼接文字和按钮，现在改成标准空状态卡片。
- 用户在未登录时也能明确知道下一步该做什么。

## 本阶段结果

- 页面主结构已从“可用”提升为“有层级、可扫描、可维护”。
- 所有主要页面已移除原来的关键内联样式，结构更适合继续迭代。
- 下一阶段进入登录弹窗、下单弹窗和反馈提示的统一优化。
