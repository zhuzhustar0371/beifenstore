### 7.7 小程序登录 500 热修

- 问题时间：2026-04-21 14:35 - 14:39
- 现象：真实小程序请求 `/api/auth/wechat-miniapp/login` 返回 500，前端旧包显示“调用微信登录失败”。
- 访问日志证据：

```text
POST /api/auth/wechat-miniapp/login HTTP/1.1 500
POST /api/auth/wechat-miniapp/login HTTP/2.0 500
```

- 根因：线上老用户已有 `users.miniapp_openid`，但不一定有 `user_wechat_auth` 记录。本次新登录逻辑先查 `user_wechat_auth`，未兼容旧 `users.miniapp_openid`，导致老用户登录时可能创建新用户，再写入同一个 `miniapp_openid`，触发唯一键冲突。
- 修复文件：`backend-api/src/main/java/com/zhixi/backend/service/UserAuthService.java`
- 修复内容：小程序登录换到 openid 后，先用 `users.miniapp_openid` 查找旧用户；如存在则补写/更新 `user_wechat_auth`，不再创建新用户。
- 重新打包：

```powershell
mvn -q -DskipTests package
```
