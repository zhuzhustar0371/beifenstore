## 2. 修改前问题诊断

### 2.1 直接故障点

- 小程序邀请页 `wechat-app/pages/invite/invite.js` 中 `qrcodeUrl` 被直接置为空字符串。
- 页面 `wechat-app/pages/invite/invite.wxml` 的二维码 `<image>` 完全依赖 `qrcodeUrl`，因此前端必然空白。

### 2.2 后端现状

- `backend-api/src/main/java/com/zhixi/backend/controller/InviteController.java` 仅提供邀请记录查询接口。
- 后端没有“邀请小程序码/邀请二维码”生成接口。
- 项目已有 `/api/auth/wechat/qr/create`，但这是网页微信登录二维码，不是小程序邀请二维码。

### 2.3 链路缺口

- 小程序登录接口 `POST /api/auth/wechat-miniapp/login` 已支持 `inviterId` 入参。
- 但小程序端未发现任何入口解析逻辑去接收分享/扫码进入时的 `inviterId` 或 `scene` 参数并写入本地缓存。
- 即使二维码图片能生成，当前版本也无法稳定完成“好友扫码进入小程序 -> 登录 -> 绑定邀请人”的闭环。

## 3. 计划修改范围

### 3.1 后端

- 为邀请模块新增小程序码生成接口。
- 在邀请服务中补充：
  - 获取微信接口 access token
  - 调用微信 `getwxacodeunlimit`
  - 生成带邀请参数的小程序码
  - 将图片二进制直接返回给小程序端下载展示

### 3.2 小程序端

- 邀请页改为真实请求二维码资源并展示。
- 增加加载态、失败态或兜底提示，避免空白区。
- 在 `app.js` 增加启动参数解析逻辑：
  - 解析 `inviterId`
  - 兼容二维码 `scene`
  - 写入本地缓存供登录时读取

## 4. 风险说明

- 微信 `getwxacodeunlimit` 依赖小程序 `appid/appsecret` 正确配置。
- 若线上环境未配置或微信接口返回错误，小程序端仍无法显示二维码，但会具备明确错误提示，不再是静默空白。
- 本次仅执行本地修复与本地检查，未执行云端构建/发布/回退；如后续需要，可在此日志继续追加发布与回退记录。

  - 调用微信 `getwxacodeunlimit`
  - 固定打开页：`pages/index/index`
  - 二维码 `scene` 参数：`inviterId=<当前用户ID>`
  - 对微信 JSON 错误响应做识别并转为业务异常

  - 文件：`wechat-app/app.js`

    - `query.scene`
    - `options.scene`
    - `referrerInfo.extraData.inviterId`

- 小程序登录页补闭环：
  - 文件：`wechat-app/pages/login/login.js`
  - 微信登录时把缓存中的 `inviterId` 透传给 `/api/auth/wechat-miniapp/login`
  - 登录成功后清理本地 `inviterId`，避免后续误绑定旧邀请人

- 小程序邀请页补真实二维码展示：

  - 使用 `wx.downloadFile` 请求 `GET /api/invites/me/qrcode`
  - 使用 `Bearer token` 下载二维码图片
  - 新增加载态、失败态、重试逻辑
