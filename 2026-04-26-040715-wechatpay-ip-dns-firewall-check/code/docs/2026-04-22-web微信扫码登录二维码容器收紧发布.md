## 2. 问题现象

- 用户反馈：微信扫码登录功能已经可用，但二维码所在的白色容器过大。
- 具体表现：
  - 二维码卡片横向铺得过宽。
  - 白色底部留白明显偏多。
  - 整个扫码区在登录弹窗内占据过大的视觉空间。

## 3. 修改前分析

### 3.1 问题定位

检查文件：

- `G:\zhiximini\zhixi-website\frontend\src\components\LoginModal.vue`

定位到的样式问题：

1. `.wechat-qr-wrap` 默认按块级元素整行铺满，导致外层白色卡片过宽。
2. `.wechat-qr-shell` 使用了较大的最小高度：
   - `min-height: 428px`
3. `.wechat-login-container` 使用微信官方 iframe 原始尺寸：
   - `300 x 400`
4. 上述样式叠加后，虽然二维码本身不大，但其所在容器显得过高、过宽。

### 3.2 处理策略

本次只调整前端样式，不改登录逻辑：

1. 白色二维码外层容器改为内容宽度自适应，并居中显示。
2. 缩小二维码展示区的高度和内边距。
3. 对微信官方 iframe 做等比缩放，减小整体视觉尺寸。
4. 保持现有微信扫码登录链路不变：
   - 后端生成 `scene`
   - 微信回调授权
   - 前端轮询状态
   - token 写入与登录态同步

### 6.1 修改文件

### 6.2 修改内容

针对二维码容器样式做了以下收紧：

1. `.wechat-qr-wrap`
   - 增加 `width: fit-content`
   - 增加 `max-width: 100%`
   - 增加 `margin: 0 auto`
   - 作用：让白色外层卡片按内容宽度收缩，并在弹窗中水平居中

2. `.wechat-qr-shell`
   - `min-height: 428px -> 380px`
   - `padding: 14px -> 10px`
   - 作用：减少二维码上下留白

3. `.wechat-login-container`
   - `width: 300px -> 270px`
   - `height/min-height: 400px -> 360px`
   - 新增 `overflow: hidden`
   - 作用：缩小微信组件外层可视区域

4. `.wechat-login-container :deep(iframe)`
   - 显式设置原始尺寸 `300 x 400`
   - 新增 `transform: scale(0.9)`
   - 新增 `transform-origin: top center`
   - 作用：把微信官方二维码 iframe 整体等比缩小

5. `.wechat-qr-placeholder`
   - `inset: 14px -> 10px`
   - 作用：和新的内边距保持一致
