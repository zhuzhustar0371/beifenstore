# 2026-04-22 16:51 登录页协议区布局修复日志

## 1. 本次目标

修复 `知禧净享生活` 小程序登录页底部“用户协议 / 隐私政策”区域的排版问题。

此前该区域使用 `navigator` 组件拼接文案，在窄屏设备上会被拆成多行块状显示，导致底部协议提示布局混乱，影响登录页观感与可读性。

## 2. 修改前状态

- 登录页底部文案为“我已阅读并同意《用户协议》和《隐私政策》”。
- 其中协议链接使用 `navigator` 组件。
- `navigator` 在当前结构下表现为块级内容，导致在手机屏幕中拆成多行显示。
- 未勾选协议时的提示文案仍只写“用户协议”，与页面实际两个协议入口不完全一致。

## 3. 备份记录

修改前已创建本地备份快照：

- `_local_backups/20260422-165139/login.wxml`
- `_local_backups/20260422-165139/login.wxss`

备份目的：

- 保留当前稳定版本。
- 出现排版或交互异常时可快速回退。

## 4. 实际修改内容

### 4.1 登录页底部协议区改为文本点击

文件：

- `wechat-app/pages/login/login.wxml`

变更：

- 将底部协议链接从 `navigator` 改为可点击的 `text`。
- 保留原有“我已阅读并同意《用户协议》和《隐私政策》”文案。
- 让文本按一句话连续排版，避免协议链接被强制拆成独立块。

### 4.2 登录页协议区样式调整

文件：

- `wechat-app/pages/login/login.wxss`

变更：

- 调整协议区容器宽度与对齐方式。
- 增加协议文本容器的 `flex-wrap`、行高设置和最大宽度限制。
- 将协议文本块收紧为居中的小块，避免横向撑满整行。
- 为协议链接增加主色和下划线样式，提升可点击识别度。
- 调整单选圆点与文本间距，减少视觉拥挤。

### 4.3 协议点击交互补充

文件：

- `wechat-app/pages/login/login.js`

变更：

- 新增 `openAgreement()` 方法，跳转到 `/pages/agreement/agreement`。
- 新增 `openPrivacy()` 方法，跳转到 `/pages/privacy/privacy`。
- 将未勾选时提示文案统一为“请先阅读并同意用户协议和隐私政策”。

## 5. 影响范围

- 仅影响登录页底部协议展示区域。
- 不修改用户协议页与隐私政策页正文内容。
- 不改动登录接口、头像上传、手机号登录等业务逻辑。
