## 现状核查

### 后端已有接口

后端已存在管理员返现打款接口：

- `POST /api/admin/cashbacks/{cashbackId}/transfer`

对应文件：

- `backend-api/src/main/java/com/zhixi/backend/controller/AdminController.java`
- `backend-api/src/main/java/com/zhixi/backend/service/AdminManageService.java`

接口行为：

- 只允许 `PENDING` 状态返现记录发起
- 查找用户的 `miniappOpenid`
- 调用微信商家转账
- 成功发起后将返现记录更新为 `PROCESSING`

## 使用说明

管理员登录后台后，在返现流水表格中：

1. 找到 `PENDING` 状态返现记录
2. 点击“批准打款”
3. 确认弹窗中的返现 ID、用户 ID、金额
4. 系统调用微信商家转账接口
5. 成功后状态应变为 `PROCESSING`

## 风险提示

- “批准打款”不是单纯审批状态，它会尝试调用微信商家转账
- 如果用户没有小程序 `openid`，后端会拒绝打款
- 如果微信商家转账权限、额度、实名校验、商户号配置或 API 权限不满足，后端会返回错误
- 当前按钮没有二次权限分级，登录管理员均可看到并点击
