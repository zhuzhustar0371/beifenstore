## 二、问题分析

1. Web 端付款并非未实现，官网已调用后端接口 `POST /api/orders/{orderId}/pay/wechat-native` 生成 Native 支付二维码。
2. 当前 Web 端付款失败的直接原因是微信支付返回：商户号未开通 Native 支付产品权限。代码已接入 Native Pay，但商户平台仍需开通产品权限。
3. 支付回调 `PayNotifyController` 原逻辑只把订单标记为已支付，并保存微信交易单号 `transactionId`，没有保存微信支付回调中的 `transaction.payer.openid`。
4. 后台商家转账逻辑需要使用与 `WECHAT_PAY_APPID` 匹配的 openid。旧逻辑在 `users.miniapp_openid` 为空时，会从 `user_wechat_auth` 中直接取用户最新微信授权记录，可能误取 Web 扫码登录 openid。
5. 生产环境仍存在 `POST /api/orders/{orderId}/pay` 模拟支付入口，认证用户可绕过微信支付直接把订单改为已支付，存在资金与返现风险。

## 四、本地修改内容

### 1. 关闭生产 mock 支付

- 文件：`backend-api/src/main/java/com/zhixi/backend/controller/OrderController.java`
- 新增配置读取：`app.pay.mock-enabled`
- `POST /api/orders/{orderId}/pay` 默认不再执行 `orderService.markPaid(orderId)`。
- 默认值为 `false`，生产环境未显式开启时返回业务错误：`生产环境不允许模拟支付`。

### 2. 增加配置项

- 文件：`backend-api/src/main/resources/application.yml`
- 新增配置：

```yaml
app:
  pay:
    mock-enabled: ${APP_PAY_MOCK_ENABLED:false}
```

### 3. 支付回调记录 payer openid

- 文件：`backend-api/src/main/java/com/zhixi/backend/controller/PayNotifyController.java`
- 支付成功回调中读取 `transaction.getPayer().getOpenid()`。
- 首次支付成功时调用 `orderService.markPaid(..., payerOpenid)`。
- 重复回调且订单已支付时，仍调用 `orderService.recordPaymentOpenid(...)` 尝试补写 openid。
- 日志只记录 `payerOpenidPresent`，不直接打印 openid。

### 4. 用户 openid 绑定逻辑

- 文件：`backend-api/src/main/java/com/zhixi/backend/service/OrderService.java`
- 新增 `recordPaymentOpenid(Long userId, String payerOpenid)`。
- 支付成功后将 payer openid 写入 `users.miniapp_openid`。
- 若 openid 已属于其他用户，则跳过绑定并记录警告，避免唯一键冲突。
- 若当前用户已有不同 `miniapp_openid`，以真实支付回调 payer openid 为准进行替换。
- 同步向 `user_wechat_auth` 写入 `source_type = MINIAPP` 的记录；如果已有 openid 记录则不重复插入。
- openid 同步失败不会回滚订单支付状态，避免微信支付回调反复失败。

### 5. 返现转账 openid 来源收紧

- 文件：`backend-api/src/main/java/com/zhixi/backend/service/AdminManageService.java`
- 商家转账仍优先使用 `users.miniapp_openid`。
- 当 `users.miniapp_openid` 为空时，只允许从 `user_wechat_auth.source_type = MINIAPP` 的记录补取 openid。
- 不再使用不区分来源的 `findByUserId`，避免误用 Web 微信扫码登录 openid。

### 6. Mapper 增强

- 文件：`backend-api/src/main/java/com/zhixi/backend/mapper/UserWechatAuthMapper.java`
- 新增：

```java
UserWechatAuth findByUserIdAndSource(Long userId, String sourceType);
```

### 7. Web Native 支付错误提示优化

- 文件：`backend-api/src/main/java/com/zhixi/backend/service/WechatPayService.java`
- Native Pay 下单失败时，如果微信返回“产品权限未开通”，后端返回更明确的业务提示：
- `网页微信扫码支付暂不可用：当前商户号未开通 Native 支付产品，请在微信支付商户平台产品中心开通后重试`
