2. 从用户提供的商户证书导出公钥

## 发现的支付初始化风险

检查日志时发现一次微信支付初始化失败记录。

微信返回核心信息：

- `RESOURCE_NOT_EXISTS`
- `无可用的平台证书，请在商户平台-API安全申请使用微信支付公钥`

判断：

- 当前代码原先使用 `RSAAutoCertificateConfig`
- 该模式会尝试从微信支付 `/v3/certificates` 自动下载平台证书
- 当前商户环境返回无可用平台证书
- 这意味着后续真实支付下单很可能需要切换到“微信支付公钥模式”

注意：

- 用户本次提供的是商户证书和 CSR，不是微信支付公钥
- 商户证书用于证明商户私钥匹配与请求签名身份
- 微信支付公钥用于验证微信支付响应/回调签名
- 两者不能混用
