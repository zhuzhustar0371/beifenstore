# 原子化待操作文档

## 目标

修复管理端表格编号类长字符串竖排问题，并优化订单管理表格比例，减少不必要横向滑动。

## 修改范围

- admin-frontend/src/views/CashbacksPage.vue
- admin-frontend/src/views/OrdersPage.vue
- admin-frontend/src/styles.css
- docs/2026-04-29-admin-table-id-columns-ellipsis-ratio-fix/README.md

## 原子步骤

1. 返现管理：给“微信批次”“微信明细”的表头和单元格加专用列类，保证 nowrap 和省略显示稳定生效。
2. 订单管理：订单号列不再使用 .transfer-cell，改为单行省略渲染。
3. 订单管理：增加订单表局部比例类，降低全局 1280px 最小宽度对订单页的影响。
4. 保持发货、退款、批准打款、确认收款、同步转账等业务接口逻辑不动。
5. 运行 npm run build，并进行本地预览冒烟验证。
