## 5. 本次修改

### 5.1 后端退款链路修正

- `backend-api/src/main/java/com/zhixi/backend/service/WechatPayService.java`
  - 为退款请求增加独立 `outRefundNo`
  - 支持单独的 `refund-notify-url`
  - 退款方法改为返回本地退款单号、微信退款单号、微信退款状态

- `backend-api/src/main/java/com/zhixi/backend/service/AdminManageService.java`
  - 将退款条件扩展为 `PAID` / `SHIPPED` / `COMPLETED`
  - 拦截重复退款提交
  - 删除错误复用发货 SQL 的逻辑
  - 微信退款时改为记录：
    - 本地退款单号
    - 微信退款单号
    - 退款状态
  - 手工退款时直接标记为退款成功并更新订单状态

- `backend-api/src/main/java/com/zhixi/backend/mapper/OrderMapper.java`
  - 订单查询增加 `refund_no`
  - 新增退款申请写入方法
  - 新增按退款单号回写退款成功/失败的方法
  - 新增正式的 `markRefunded(...)`

- `backend-api/src/main/java/com/zhixi/backend/model/Order.java`
  - 新增 `refundNo`

- `backend-api/src/main/java/com/zhixi/backend/service/OrderService.java`
  - 新增按退款单号处理退款成功/失败回调的方法

### 5.2 微信退款回调

- `backend-api/src/main/java/com/zhixi/backend/controller/PayNotifyController.java`
  - 保留原支付回调
  - 新增退款回调：
    - `POST /api/pay/wechat/refund/notify`
  - 使用微信支付 SDK 解析 `RefundNotification`
  - 根据退款回调状态回写订单：
    - `SUCCESS` -> 订单状态改为 `REFUNDED`
    - `CLOSED` / `ABNORMAL` -> 退款状态改为 `FAILED`
