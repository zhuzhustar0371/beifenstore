# 2026-04-30 偏远地区运费与返现规则改造原子化说明

## 目标范围

1. 偏远地区固定为：香港、澳门、台湾、青海、西藏、新疆、内蒙古。
2. 命中偏远地区的订单加收运费 `20.00` 元。
3. 默认收货地址命中偏远地区时，下单前必须提示。
4. 运费不参与自购返现和邀请返现。
5. 自购返现规则：
   - 第 1 单 `0%`
   - 第 2 单 `10%`
   - 第 3 单 `20%`
   - 第 4 单 `100%`
   - 第 5 单及以后 `0%`
6. 邀请返现规则：
   - 每满 3 个有效首单算一批
   - 第 1 批 `100%`
   - 第 2 批及以后每批 `20%`

## 原子步骤

### A. 备份基线

- 创建本地备份目录和操作文档
- 复制当前工作区完整快照到 `G:\store`
- 同步同一份快照到 `beifenstore.git` 备份分支

### B. 订单金额模型拆分

- 在订单模型中新增：
  - `productAmount`
  - `shippingFee`
  - `cashbackBaseAmount`
  - `province`
  - `city`
  - `district`
- 保持 `totalAmount` 表示实付款

### C. 下单运费接入

- 小程序下单接口补传 `province/city/district`
- 后端按省份识别偏远地区
- 创建订单时计算：
  - 商品金额
  - 运费
  - 返现基数
  - 实付款

### D. 自购返现改造

- 返现比例保持不变
- 返现基数改为 `cashbackBaseAmount`
- 退款重算时也按 `cashbackBaseAmount` 处理

### E. 邀请返现改造

- 停用旧的逐人首单返现
- 启用每满 3 人一批的返现
- 退款导致首单失效时，撤销超出的邀请批次返现
- 允许已取消批次在后续重新满足条件时再次触发

### F. 小程序展示改造

- 默认地址命中偏远地区时显示提示
- 地区切换命中偏远地区时实时提示
- 订单详情展示商品金额、运费、返现基数、实付款
- 规则页默认文案同步到最新规则

### G. 数据迁移与兼容

- 数据库迁移增加新字段
- 历史订单回填：
  - `productAmount = totalAmount`
  - `shippingFee = 0`
  - `cashbackBaseAmount = productAmount`

### H. 验证与留档

- 后端执行 Maven 编译
- 小程序关键脚本执行 `node --check`
- 补齐本次执行日志、备份位置、回退依据
