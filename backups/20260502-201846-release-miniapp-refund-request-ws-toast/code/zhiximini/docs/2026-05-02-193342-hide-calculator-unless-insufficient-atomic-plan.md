# 提现页点击申请提现后再显示计算器原子化方案

- 时间戳：20260502-193342
- 目标页面：`wechat-app/pages/cashback/cashback`
- 目标：页面默认隐藏“提现计算器”区块；只有用户点击“申请提现”且当前方案存在待扣款不足时，才显示该区块。

## 变更范围

1. `wechat-app/pages/cashback/cashback.wxml`
2. `docs/2026-05-02-193342-hide-calculator-unless-insufficient-implementation-log.md`

## 原子化步骤

1. 备份当前 `wechat-app` 到本地 `G:\store` 和远端备份仓 `beifenstore`。
2. 在 `cashback.js` 增加前端显示开关，默认隐藏“提现计算器”。
3. 在点击 `申请提现` 时：
   - 若当前不可申请且存在待扣款不足，打开“提现计算器”区块
   - 其余情况不显示该区块
4. 将 `cashback.wxml` 中“提现计算器”外层 `list-section` 改为依赖显示开关和 `hasRemainingDebt` 的条件渲染。
5. 保留顶部汇总卡与返现明细列表，不影响提现提交逻辑。
6. 自检模板结构与按钮逻辑，确保页面渲染正常。
7. 上传新的体验版供验证。

## 验收标准

1. 页面初始加载时，看不到“提现计算器”整块。
2. 点击“申请提现”且当前存在待扣款不足时，页面才显示“提现计算器”整块。
3. 点击“申请提现”且可正常提现时，继续走原有流程，不额外常驻显示该区块。
4. 顶部汇总卡、申请提现按钮、返现明细列表保持可用。
