## 4. 本地代码修改（已执行）
### 4.1 小程序端
1. `wechat-app/app.json`
- 新增 `plugins.loginplugin` 配置：
  - `version: latest`
  - `provider: wx12251485dfaf24d3`

2. `wechat-app/pages/login/login.json`
- 新增页面组件引用：
  - `"login": "plugin://loginplugin/login"`

3. `wechat-app/pages/login/login.wxml`
- 重构登录页结构并挂载插件组件：
  - `<login show="{{login_show}}" modal="{{modal}}" bindsuccess="loginSuccess" bindfail="loginFail" bindcancel="loginCancel" />`

4. `wechat-app/pages/login/login.js`
- 新增插件控制数据：`login_show`、`modal`。
- 登录入口改为弹出插件并在回调中完成业务登录：
  - `loginSuccess`：读取插件返回的 `nickName/avatarUrl` 并提交后端登录；
  - `loginFail`：按兜底策略继续登录（空头像昵称，由后端默认）；
  - `loginCancel`：关闭弹窗。
- 保留 `wx.login` 获取 code，提交字段：`code + inviterId + nickName + avatarUrl`。
- 登录成功写缓存时补充头像兜底：`resData.avatarUrl || profile.avatarUrl`。

5. `wechat-app/pages/user/user.wxml`
- 头像展示改为优先使用 `userInfo.avatarUrl`，无值时回退默认头像。

6. `wechat-app/pages/user/user.js`
- `refreshUserProfile` 的 `nextUserInfo` 新增 `avatarUrl` 合并逻辑：
  - `data.avatarUrl || currentUserInfo.avatarUrl || ''`

### 4.2 后端
1. `backend-api/src/main/java/com/zhixi/backend/service/UserAuthService.java`
- 新增方法：`getUserAvatarUrl(Long userId)`，从 `user_wechat_auth` 读取头像。
- 登录返回 `authResponse` 新增字段：`avatarUrl`。

2. `backend-api/src/main/java/com/zhixi/backend/controller/AuthController.java`
- `/api/auth/me` 返回新增字段：`avatarUrl`。
