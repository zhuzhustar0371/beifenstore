## 关键排查发现

### 发现 1：线上后端虽已写入 AppSecret，但最初仍在跑旧 jar

在第一次验证小程序登录接口时，出现了异常现象：

- 向 `/api/auth/wechat-miniapp/login` 传入明显伪造的 `code`
- 接口却成功创建用户并返回 token

这与微信官方接口行为不一致。为验证这一点，又直接请求微信官方：

- `https://api.weixin.qq.com/sns/jscode2session?...`

微信官方返回：

- `errcode=40029`
- `errmsg=invalid code`

结论：

- 线上接口成功并不是因为微信接受了假 `code`
- 而是服务器当时运行的 `app.jar` 仍是旧版本逻辑

### 发现 2：本地源码已经是新逻辑，但服务器 jar 时间明显更早

核查结果：

- 本地源码 `backend-api/src/main/java/com/zhixi/backend/service/UserAuthService.java`
  - 修改时间：`2026-04-20 15:39:42`
- 服务器 `app.jar`
  - 修改时间：`2026-04-19 23:03`

结论：

- 仓库里的真实 `jscode2session` 逻辑并没有真正部署到线上
- 仅补环境变量还不够，必须把最新后端 jar 一并上线
