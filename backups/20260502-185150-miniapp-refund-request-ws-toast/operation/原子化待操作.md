# 原子化待操作 — 小程序退款申请 + 后台审批 + WebSocket 提醒

## 时间戳
20260502-185150

## 概述
为小程序订单详情页增加退款申请入口，引入"用户申请 → 后台审批 → 执行退款"的二阶段流程。

## 核心原则
1. refund_request_status（申请状态）与 refund_status（真实退款状态）完全分离
2. 复用现有 `/ws/withdrawals` WebSocket 通道
3. 管理端全局右下角小浮层提醒，非阻塞式
4. 小程序首版仅订单详情页做申请，列表页仅显示状态
5. 首版直接在 orders 表加字段，不建新表


状态枚举：NONE / PENDING / APPROVED / REJECTED

## 新增接口

### 用户侧
- `POST /api/orders/{orderId}/refund-request` — 提交退款申请
  - 请求体: { reason: string }
  - 校验: 订单归属、状态、无重复申请、reason 非空

### 管理端
- `POST /api/admin/orders/{orderId}/refund/approve` — 批准退款
  - 请求体: { amount: number, adminRemark: string }
- `POST /api/admin/orders/{orderId}/refund/reject` — 驳回申请
  - 请求体: { reviewRemark: string }

## WebSocket 事件
- 事件名: `refund-request-created`
- Payload: { orderId, orderNo, userId, userNickname, refundRequestReason, refundRequestAt, refundRequestStatus }
