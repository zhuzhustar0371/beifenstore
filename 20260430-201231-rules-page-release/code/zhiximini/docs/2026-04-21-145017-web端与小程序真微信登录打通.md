## 开发前分析

- 当前小程序端 `wechat-app/pages/login/login.js` 已通过 `wx.login` 获取 code，并调用 `/api/auth/wechat-miniapp/login`。
- 当前后端 `backend-api/src/main/java/com/zhixi/backend/service/UserAuthService.java` 已存在 `jscode2session` 调用，可换取真实 miniapp openid。
- 当前 Web 端 `zhixi-website/frontend/src/components/LoginModal.vue` 只有手机号密码登录和注册入口，没有微信小程序扫码登录 UI。
- 当前后端 Web 微信二维码逻辑位于 `/api/auth/wechat/qr/create`，生成的是开放平台网页扫码 URL，并且存在 `/api/auth/wechat/mock-scan` 模拟扫码接口，不满足“Web 端和小程序端打通真正微信登录”的目标。
- 实施方案：Web 端展示后端生成的小程序码；用户用微信扫码进入小程序确认页；小程序确认页通过 `wx.login` 获取 code；后端用 code 换真实 openid，创建/复用用户，授权二维码会话；Web 端轮询拿到 token 完成登录。
