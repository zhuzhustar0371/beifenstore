# 重置所有用户返现统计

## 日期
2026-04-24

## 需求
重置所有用户的返现统计：
1. **个人返现顺序** — 用户之前的下单记录不再计入顺序，从第1单重新算起
2. **邀请批次返现** — 清空所有已统计的首单记录，重新开始计数
3. **已有返现记录** — 取消所有待结算的返现

## 实现方案

### 核心思路：cashback_reset_at

在 `users` 表新增 `cashback_reset_at` 字段。下单顺序计数 SQL 增加条件：只统计 `pay_time > cashback_reset_at` 的订单。

### 重置逻辑

POST /api/admin/cashbacks/reset-all:
1. 所有用户 `cashback_reset_at = NOW()`
2. 所有 `PENDING` 返现 → `CANCELLED`
3. 所有邀请 `first_paid_time = NULL`
