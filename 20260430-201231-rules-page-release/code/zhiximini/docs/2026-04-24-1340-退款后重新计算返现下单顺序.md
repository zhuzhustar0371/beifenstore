# 退款后重新计算个人返现下单顺序

## 日期
2026-04-24

## 需求
退款后需重新计算用户剩余有效订单的下单顺序，按新顺序调整个人返现金额。

### 返现规则
| 下单顺序 | 返现比例 |
|---------|---------|
| 第1单 | 不返 |
| 第2单 | 10% |
| 第3单 | 20% |
| 第4单 | 70% |
| 第5单及以后 | 不返 |

### 场景示例
用户有 3 单已付款：
- 订单1：第1单 → 不返
- 订单2：第2单 → 返 10%
- 订单3：第3单 → 返 20%

如果订单2退款：
- 当前代码：取消订单2的返现 ✅
- 遗漏：订单3现在应该是第2单 → 应返 10%（而不是 20%）❌

## 修复内容

### 修改文件

| 文件 | 改动 |
|------|------|
| `OrderMapper.java` | 新增 `findValidPaidByUserOrderByIdAsc()` 查询有效已付款订单按ID升序 |
| `CashbackService.java` | 抽取 `calculatePersonalCashbackAmount()` 方法 |
| `OrderService.java` | 新增 `recalculatePersonalCashbackAfterRefund()` + 在 `cancelCashbackForRefundIfNeeded()` 中调用 |

### 修复逻辑

`recalculatePersonalCashbackAfterRefund(userId)`：

1. 查询用户所有有效已付款（非退款）订单，按 ID 升序排列
2. 对每个订单按新位置计算预期返现金额
3. 对比现有返现记录：
   - 预期不返 + 现有记录 → 取消现有记录
   - 预期返现 + 无记录 → 创建新记录
   - 预期返现 + 金额不一致 → 取消旧记录 + 创建新记录
   - 预期返现 + 金额一致 → 不做改动

### 调用时机

在 `cancelCashbackForRefundIfNeeded()` 中，取消完退款订单本身的返现后，立即调用重新计算。
