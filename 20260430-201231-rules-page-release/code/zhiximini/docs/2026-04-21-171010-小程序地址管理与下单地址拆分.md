## 一、变更前分析

1. `wechat-app/pages/address-edit/address-edit.js` 的 `save()` 固定调用 `/api/orders` 创建订单。
2. `wechat-app/pages/user/user.js` 的“管理收货地址”入口直接跳到 `address-edit`，未进入地址簿页。
3. 后端无独立地址表，仅 `orders` 表中有收货字段，无法支撑“地址管理独立保存 + 下单复用”。
4. 结论: 必须补 `user_addresses` 表和地址 CRUD 接口，并将前端入口按场景分流。

## 三、本地修改记录

### 3.1 后端: 新增地址表与地址接口

- 变更文件:
  - `backend-api/src/main/resources/schema.sql`
  - `backend-api/src/main/java/com/zhixi/backend/config/DatabaseMigrationRunner.java`
  - `backend-api/src/main/java/com/zhixi/backend/model/UserAddress.java` (新增)
  - `backend-api/src/main/java/com/zhixi/backend/dto/SaveUserAddressRequest.java` (新增)
  - `backend-api/src/main/java/com/zhixi/backend/mapper/UserAddressMapper.java` (新增)
  - `backend-api/src/main/java/com/zhixi/backend/service/UserAddressService.java` (新增)
  - `backend-api/src/main/java/com/zhixi/backend/controller/AddressController.java` (新增)

- 实施内容:
  1. 在 `schema.sql` 新增 `user_addresses` 表结构:
     - 字段: `recipient_name / recipient_phone / province / city / district / detail_address / is_default`
     - 关联字段: `user_id`
     - 索引: `idx_user_addresses_user_id`
  2. 在 `DatabaseMigrationRunner` 增加 `ensureTable("user_addresses", ...)`，保证线上已有库可自动建表。
  3. 新增地址 CRUD 接口 `/api/addresses`:
     - `GET /api/addresses` 获取当前用户地址列表
     - `GET /api/addresses/default` 获取默认地址（若无默认则返回优先地址）
     - `GET /api/addresses/{id}` 获取地址详情（带用户归属校验）
     - `POST /api/addresses` 新增地址
     - `PUT /api/addresses/{id}` 更新地址
     - `DELETE /api/addresses/{id}` 删除地址
  4. 默认地址策略:
     - 首次新增地址自动设为默认
     - 保存为默认时，自动清理同用户其他默认
     - 删除默认地址后，自动将剩余优先地址设为默认

### 3.2 小程序: 地址管理与下单地址分流

- 变更文件:
  - `wechat-app/pages/user/user.js`
  - `wechat-app/pages/product/product.js`
  - `wechat-app/pages/address/address.js`
  - `wechat-app/pages/address-edit/address-edit.js`

- 实施内容:
  1. “我的”页地址入口从 `address-edit` 改为地址列表页:
     - `goAddress()` -> `/pages/address/address`
  2. 商品下单进入地址页时显式标记场景:
     - `/pages/address-edit/address-edit?scene=order&productId=...&quantity=...`
  3. 地址列表页接入真实地址接口:
     - 拉取列表、编辑、删除都对接 `/api/addresses`
  4. 地址编辑页引入场景分流:
     - `scene=manage`: 仅保存地址，不创建订单
     - `scene=order`: 先保存地址，再创建订单并拉起支付
  5. 二次购买自动回填:
     - `scene=order` 且未指定 `id` 时，自动请求 `/api/addresses/default` 回填表单
  6. 地址统一来源:
     - 管理地址与下单地址均写入/读取同一地址表 `user_addresses`
