## 用户现场问题

用户截图中显示：

- 页面：微信小程序“我的”页
- 错误：`Unknown column 'out_batch_no' in 'field list'`
- 涉及 SQL：查询 `cashback_records`
- 影响：进入“我的”页时，返现统计接口触发 SQL 异常，错误信息直接覆盖到小程序界面上

用户同时反馈：

- “登录进去的还是模拟登录”

## 排查结论

### 1. 报错根因

线上数据库 `cashback_records` 表仍为旧结构，实际字段只有：

- `id`
- `user_id`
- `cashback_type`
- `related_order_id`
- `related_invite_batch_id`
- `amount`
- `status`
- `remark`
- `created_at`

但当前后端 `CashbackRecordMapper` 查询已经使用新字段：

- `out_batch_no`
- `out_detail_no`
- `transfer_id`
- `transfer_time`

因此访问 `/api/cashbacks/{userId}` 时会触发 SQL 异常。

### 2. “像模拟登录”的根因

线上最新验证显示：

- 真实小程序登录已经创建了真实 `openid` 用户
- 最新用户 `miniapp_openid` 形如 `oChm...`，这是微信小程序真实 `openid` 格式
- 假 `code` 调用 `/api/auth/wechat-miniapp/login` 已返回微信官方 `invalid code`

但页面看起来像模拟登录，主要有两个原因：

- 历史旧 jar 曾创建过 `mock_mini_openid_...` 测试用户，数据库里仍能看到旧记录
- 新小程序用户没有绑定手机号时，后端会生成内部占位手机号，前端原来直接把这个占位手机号展示在“我的”页

本次修复重点是：

- 不再让“我的”页展示内部占位手机号
- 改为展示邀请码或“微信已授权登录”
- 刷新本地缓存中的用户信息，避免旧缓存继续显示旧数据

## 本次代码修改

### 1. 后端自动迁移补齐返现打款字段

修改文件：

- `backend-api/src/main/java/com/zhixi/backend/config/DatabaseMigrationRunner.java`

新增迁移项：

- `cashback_records.out_batch_no`
- `cashback_records.out_detail_no`
- `cashback_records.transfer_id`
- `cashback_records.transfer_time`

目的：

- 让已有线上旧表在应用启动时自动补齐当前 Mapper 依赖字段
- 避免再次出现 `Unknown column 'out_batch_no'` 报错

### 2. 后端未处理异常不再向前端暴露原始 SQL

修改文件：

- `backend-api/src/main/java/com/zhixi/backend/common/GlobalExceptionHandler.java`

修改内容：

- 未处理异常会在服务端日志里完整记录
- 返回给前端的消息统一改为：`服务开小差了，请稍后重试`

目的：

- 防止 SQL、堆栈、表结构等技术细节直接展示给小程序用户
- 即使后续出现未知异常，也不再铺满一整屏数据库错误

### 3. `/api/auth/me` 兼容返回 `userId`

修改文件：

- `backend-api/src/main/java/com/zhixi/backend/controller/AuthController.java`

修改内容：

- 在原有 `id` 基础上新增 `userId`

目的：

- 小程序端刷新用户缓存时可直接复用登录接口中的字段结构
- 减少前端对 `id/userId` 差异的兼容成本

### 4. 小程序“我的”页刷新真实用户资料

修改文件：

- `wechat-app/pages/user/user.js`

修改内容：

- 进入“我的”页时，如果本地有 token，会调用 `/api/auth/me`
- 用服务端返回的真实用户资料刷新本地 `userInfo`
- 如果 token 失效，会清理本地用户缓存并重置统计
- 返现与邀请统计在用户资料刷新后再请求

目的：

- 避免旧缓存导致用户看到历史 mock 信息
- 让小程序端展示与后端当前登录态保持一致

### 5. 小程序“我的”页不再展示内部占位手机号

修改文件：

- `wechat-app/pages/user/user.wxml`
- `wechat-app/pages/user/user.wxss`

修改内容：

- 移除头像卡片中的手机号展示
- 登录后优先展示：`邀请码 xxx`
- 无邀请码时展示：`微信已授权登录`

目的：

- 避免后端内部生成的占位手机号被用户误认为“模拟登录”
- 更符合小程序微信授权登录后的用户感知

### 6. 小程序请求层过滤技术错误信息

修改文件：

- `wechat-app/utils/request.js`

修改内容：

- 当后端返回包含 SQL、Exception、`###` 等技术关键词的错误消息时，小程序端显示通用失败提示

目的：

- 前端兜底防止技术错误直接展示给用户
- 与后端异常收敛形成双保险
