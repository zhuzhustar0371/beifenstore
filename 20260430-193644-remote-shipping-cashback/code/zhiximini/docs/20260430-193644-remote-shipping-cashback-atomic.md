# 2026-04-30 偏远地区运费与返现规则改造原子化说明

## 目标范围

1. 偏远地区识别范围固定为：港澳台、青海、西藏、新疆、内蒙古。
2. 命中偏远地区订单增加运费 `20.00` 元。
3. 默认收货地址若命中偏远地区，下单前明确提示。
4. 运费不参与任何自购返现与邀请返现计算。
5. 返现规则统一为：
   - 自购：第 1 单 `0%`，第 2 单 `10%`，第 3 单 `20%`，第 4 单 `100%`，第 5 单及以后 `0%`
   - 邀请：每满 3 个有效首单结算一批，第 1 批 `100%`，第 2 批及以后每批 `20%`

## 原子化拆分

### 原子步骤 A：备份基线

- 创建本地备份目录与说明文档
- 复制当前代码快照
- 推送同内容到远端备份仓库

### 原子步骤 B：后端订单金额模型修正

- 增加订单中的商品金额、运费、支付金额、返现基数字段
- 调整订单查询映射
- 保证历史兼容读取

### 原子步骤 C：后端下单运费规则接入

- 订单创建请求补充结构化地区字段
- 服务端判定偏远地区并计算 `shippingFee`
- 生成 `payAmount` 与 `cashbackBaseAmount`

### 原子步骤 D：后端返现逻辑统一

- 自购返现改为基于 `cashbackBaseAmount`
- 邀请返现切换为“每满 3 人一批”
- 移除或停用“每个首单直接返 100%”的现行接线

### 原子步骤 E：小程序下单与提示修正

- 默认地址命中偏远地区时提示
- 编辑地址切换到偏远地区时提示
- 下单提交传递结构化地址字段
- 订单详情展示商品金额、运费、实付金额，并标注运费不返现

### 原子步骤 F：官网/后台展示口径统一

- 规则页和首页返现说明统一
- 后台订单金额展示补充分项

### 原子步骤 G：验证与归档

- 验证偏远地区与普通地区下单金额差异
- 验证返现基数不含运费
- 验证邀请批次返现触发口径
- 补充完整执行结果、风险与回退依据

## 回退原则

1. 若本地改造失败，保留原仓库现状并停止继续提交。
2. 若远端构建或发布失败，必须使用本次备份快照恢复。
3. 回退依据优先使用本次本地备份与 `beifenstore.git` 备份仓库中的同时间戳目录。
