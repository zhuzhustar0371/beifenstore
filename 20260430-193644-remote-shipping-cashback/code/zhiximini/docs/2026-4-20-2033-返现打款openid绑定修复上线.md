# 返现打款 openid 绑定修复上线

## 记录时间

- 时间：2026-04-20 20:33（Asia/Shanghai）
- 操作人：Codex
- 目标：修复管理后台返现打款时提示“该用户不存在或尚未绑定微信小程序”的问题。

## 问题现象

线上管理后台点击“批准打款”时弹出错误：

- `该用户不存在或尚未绑定微信小程序，无法进行微信打款`

页面中可见返现记录：

- 用户 ID：`9`
- 状态：`PENDING`

## 原因分析

本次核查后发现，这不是单一的数据问题，后端绑定链路本身存在两个缺口：

### 1. 管理端打款只读取 `users.miniapp_openid`

原逻辑位于：

- `backend-api/src/main/java/com/zhixi/backend/service/AdminManageService.java`

问题：

- 打款时只检查 `users.miniapp_openid`
- 如果该字段为空，就直接报“未绑定微信小程序”
- 但系统里其实还有另一份微信绑定数据表：
  - `user_wechat_auth`

这意味着：

- 用户可能已经做过微信绑定
- 但 `users` 表里的 `miniapp_openid` 没同步
- 管理台打款仍然会误判为“未绑定”

### 2. 绑定手机流程没有把 openid 同步回 `users`

原逻辑位于：

- `backend-api/src/main/java/com/zhixi/backend/service/UserAuthService.java`

问题：

- `bindWechatMobile(...)` 会往 `user_wechat_auth` 写入 openid
- 但不会更新 `users.miniapp_openid`

结果：

- 绑定完成后，用户侧登录链路可能可用
- 管理台返现打款仍可能失败

### 3. 小程序登录优先按 `users.miniapp_openid` 查用户

问题：

- `loginByMiniapp(...)` 原本只按 `users.miniapp_openid` 查找
- 如果没查到，就直接创建占位手机号新用户
- 没有先去 `user_wechat_auth` 里查是否已有同一个 openid 绑定过其它账号

结果：

- 已绑定微信的老用户可能无法被正确复用
- 会增加用户与 openid 关系分裂的风险

## 本次修复

### 1. 新增按用户 ID 查询微信绑定

修改文件：

- `backend-api/src/main/java/com/zhixi/backend/mapper/UserWechatAuthMapper.java`

新增方法：

- `findByUserId(Long userId)`

用途：

- 管理后台打款时，当 `users.miniapp_openid` 为空，可回退查 `user_wechat_auth`

### 2. 管理台打款支持从 `user_wechat_auth` 回填 openid

修改文件：

- `backend-api/src/main/java/com/zhixi/backend/service/AdminManageService.java`

修改内容：

- 新增 `resolveCashbackOpenid(User user)`：
  - 先读 `users.miniapp_openid`
  - 若为空，则查 `user_wechat_auth`
  - 若查到 openid，则自动补写回 `users.miniapp_openid`
- `transferCashback(...)` 改为使用该方法解析 openid

结果：

- 旧数据用户只要在 `user_wechat_auth` 中存在 openid
- 即可继续完成返现打款
- 并且会顺手修复 `users.miniapp_openid`

### 3. 扫码绑手机时同步写入 `users.miniapp_openid`

修改文件：

- `backend-api/src/main/java/com/zhixi/backend/service/UserAuthService.java`

修改内容：

- 在 `bindWechatMobile(...)` 中新增：
  - `userService.updateMiniappOpenid(user.getId(), session.getOpenid())`

结果：

- 以后用户完成扫码绑定手机号时
- `users` 表和 `user_wechat_auth` 会同时保留 openid

### 4. 小程序登录优先复用已绑定账号

修改文件：

- `backend-api/src/main/java/com/zhixi/backend/service/UserAuthService.java`

修改内容：

- `loginByMiniapp(...)` 现在按以下顺序找用户：
  1. 先查 `users.miniapp_openid`
  2. 若未命中，再查 `user_wechat_auth.openid`
  3. 若命中，则把 openid 回填到 `users.miniapp_openid`
  4. 仍未命中才创建占位用户
- 同时，如果找到了用户但 `users.miniapp_openid` 为空，也会自动补齐

结果：

- 避免同一个微信 openid 被错误拆成新老两个账号
- 旧绑定账号可被小程序登录正确复用

## 本次结论

本次已上线修复返现打款读取 openid 的逻辑缺口。

现在有两种情况：

1. 如果用户 `9` 在 `user_wechat_auth` 中已经有 openid  
   那么再次点击“批准打款”时，系统会自动补齐 `users.miniapp_openid`，并继续发起微信打款。

2. 如果用户 `9` 在 `user_wechat_auth` 和 `users.miniapp_openid` 中都没有 openid  
   那么这条报错仍会出现，这说明该用户确实没有完成过微信小程序授权或绑定，需要先让该用户登录一次小程序。
