## 产品概述

修复管理端"批准打款"功能，使管理员能够正常向用户发起微信商家转账。当前后端和前端代码均已实现转账功能，但线上环境变量和证书配置可能未正确设置，导致功能不可用。

## 核心功能

- 配置微信支付转账所需的全部密钥和证书到线上环境
- 确保管理端"批准打款"按钮能正常调用微信支付API完成转账
- 支持转账状态同步和失败重试
- 操作前完整备份所有相关代码和配置

## 技术栈

- 后端：Java Spring Boot + 微信支付 Java SDK (wechat-pay-java)
- 前端：Vue 3 + Axios
- 配置管理：环境变量（.env文件）+ application.yml

## 实施方案

通过配置线上环境变量和部署证书文件，使现有的转账代码能够正常调用微信支付API。无需修改代码，只需正确配置即可。

### 关键配置项

根据 `application.yml` 和环境变量映射，需要设置：

| 环境变量 | 值 | 说明 |
| --- | --- | --- |
| WECHAT_PAY_API_V3_KEY | eoHdm38s2k6x94kVJ96wU5bGaF63aHjS | APIv3密钥 |
| WECHAT_PAY_SERIAL_NUMBER | 43556111DFA04364EEF593A44815B6A556F2BDDA | 商户证书序列号 |
| WECHAT_PAY_PUBLIC_KEY_ID | PUB_KEY_ID_0111072384812026041400292210003804 | 微信支付公钥ID |
| WECHAT_PAY_TRANSFER_SCENE_ID | 1000 | 转账场景ID（现金营销） |
| WECHAT_PAY_TRANSFER_USER_RECV_PERCEPTION | 活动奖励 | 用户收款感知 |


### 证书文件部署

需要部署到服务器 `/home/ubuntu/apps/backend-api/cert/` 目录：

1. `apiclient_key.pem` - 商户私钥
2. `apiclient_cert.pem` - 商户证书
3. `wechatpay_public_key.pem` - 微信支付公钥

### 实施步骤

1. 备份项目代码和配置
2. 将证书文件部署到服务器
3. 更新线上 `.env` 文件
4. 重启后端服务
5. 验证转账功能

## 架构设计

当前架构已完整支持转账功能：

- `WechatPayService.transferToWallet()` - 调用微信支付API
- `AdminManageService.transferCashback()` - 业务逻辑处理
- `AdminController.transferCashback()` - 提供REST API
- 前端 `CashbacksPage.vue` - "批准打款"按钮和状态同步

无需修改架构，只需正确配置环境变量指向证书文件。
