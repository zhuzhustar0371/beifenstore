# 管理端动效优化

## 日期
2026-04-24

## 改动概要
在现有管理端前端项目中增加三类动效，提升界面交互品质。**不增加新文件，不修改后端**。

## 改动文件

| 文件 | 改动类型 | 说明 |
|------|----------|------|
| `tailwind.config.js` | 修改 | 新增 3 组关键帧 + 3 个动画工具类 |
| `styles.css` | 修改 | 新增按钮` :active` 脉冲、侧边栏 hover 增强、卡片入场动画 |
| `MetricCard.vue` | 修改 | 新增数字滚动计数（`countTo` / `countPrefix` / `countSuffix` props） |
| `OverviewPage.vue` | 修改 | MetricCard 调用传入原始数值触发计数动画 |
| `MainLayout.vue` | 修改 | 侧边栏菜单交错淡入入场动画 |

## 动效详情

### 1. 按钮点击波纹/发光反馈
- `tailwind.config.js`：新增 `btn-pulse` 关键帧 — box-shadow 从 0 扩散到 18px 消失
- `styles.css`：所有按钮类（`.btn-primary`、`.btn-secondary`、`.btn-inline`、`.icon-button`）的 `:active` 状态触发 `btn-pulse` 动画 + 轻微 scale 缩小
- `.icon-button` 的 `:active` 额外触发图标弹跳旋转

### 2. 数字滚动计数
- `MetricCard.vue`：新增 `countTo`（目标数值）、`countPrefix`（前缀如 ¥）、`countSuffix`（后缀）props
- 使用 `requestAnimationFrame` + cubic easeOut 缓动，900ms 从 0 滚动到目标值
- 数据刷新时自动重新触发动画
- 未传入 `countTo` 时兼容原有 `value` 显示逻辑

### 3. 侧边栏 hover 增强
- `styles.css`：`.sidebar-link:hover` 时：
  - 左侧指示光柱（`::before`）半透明发光（opacity: 0.5 + 增强 blur）
  - 图标触发弹跳旋转动画（`sidebar-icon-hover`）
  - 文字 letter-spacing 微增 0.015em
- 活跃态（`.router-link-active`）图标增加 drop-shadow 光晕

### 4. 额外 — 卡片淡入入场
- `styles.css`：`.glass-card` 默认应用 `fade-in-up` 动画，按 nth-child 顺序延迟 0/80/160/240/320/400ms
- `MainLayout.vue`：侧边栏菜单项按 index 依次延迟 60ms 淡入
