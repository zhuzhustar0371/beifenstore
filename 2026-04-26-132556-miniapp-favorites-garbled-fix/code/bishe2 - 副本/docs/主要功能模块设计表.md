## 主要功能模块设计表（三线表）

表 3-1 主要功能模块设计表

| 功能模块 | 子功能 | 主要功能说明 |
| --- | --- | --- |
| 用户认证模块 | Web 账号登录、微信授权登录、手机号绑定、密码找回 | 实现普通用户身份认证与登录授权，支持 Web 端账号密码登录、小程序微信授权登录、手机号绑定及短信验证码找回密码，统一维护用户会话状态。 |
| 首页浏览模块 | 信息流展示、分类筛选、地区筛选、关键词搜索 | 展示平台帖子信息，支持按交易类型、商品分类、行政区划和关键词进行检索，帮助用户快速查找本地二手商品。 |
| 帖子发布模块 | 出售发布、求购发布、图片上传、表单校验 | 支持用户发布出售帖和求购帖，填写标题、描述、价格、地区等信息，并上传商品图片，提交后进入审核流程。 |
| 帖子详情模块 | 帖子详情展示、卖家信息展示、收藏操作、状态展示 | 展示帖子标题、价格、图片、描述和发布者信息，支持用户收藏或取消收藏，并展示帖子审核与上下架状态。 |
| 收藏管理模块 | 收藏切换、收藏列表、分页展示 | 记录用户收藏的帖子信息，支持收藏状态切换、收藏列表查询和分页展示，便于用户回看感兴趣商品。 |
| 会话消息模块 | 发起会话、消息列表、文本消息、图片消息、已读处理 | 用户可围绕帖子发起会话，查看会话列表和历史消息，发送文本、图片等消息，并对未读消息进行同步处理。 |
| 订单预留模块 | 聊天下单、订单记录生成、状态记录 | 在聊天场景中根据帖子生成订单记录，保存买卖双方、商品标题、价格和订单状态，为后续交易闭环扩展预留接口。 |
| 文件上传模块 | 聊天图片上传、帖子图片上传、文件类型校验 | 对聊天图片和商品图片上传进行统一处理，限制文件大小与格式，并生成可访问的资源地址供前端展示。 |
| 用户中心模块 | 个人信息展示、我的发布、我的收藏、登录状态维护 | 为用户提供个人主页入口，集中展示个人资料、已发布帖子、收藏内容和账户状态，提升自助管理能力。 |
| 后台审核模块 | 帖子审核、驳回、下架、恢复上架 | 管理员对用户发布的出售帖和求购帖进行审核，支持通过、驳回、下架和恢复操作，保障平台内容合规。 |
| 用户管理模块 | 用户查询、禁用启用、角色分配、密码重置 | 管理员可按状态和角色检索用户，对违规用户执行禁用或恢复操作，并可分配客服角色或重置账号密码。 |
| 客服工作台模块 | 会话跟进、客服回复、处理标记、退款/驳回预留入口 | 客服人员可在工作台查看待处理会话、回复用户消息、标记处理状态，并预留退款与驳回等平台治理扩展能力。 |
| 反馈处理模块 | 反馈查看、状态流转、问题闭环 | 管理员和客服可查看用户反馈信息，并对反馈状态进行“待处理、处理中、已关闭”的闭环管理。 |
| 实时通信模块 | WebSocket 推送、消息同步、已读通知 | 系统通过 WebSocket 推送新消息和会话已读状态，提升站内沟通的实时性与交互体验。 |
| 基础数据模块 | 区县数据管理、用户数据管理、帖子数据管理 | 负责维护用户、帖子、会话、消息、反馈、区县等核心数据集合，为前台展示与后台治理提供数据支撑。 |

## Word 三线表排版建议

复制到 Word 后，可按以下方式设置为三线表：

1. 保留表头行。
2. 仅保留表格顶线、表头下边线、表格底线。
3. 取消其余内部竖线和横线。
