# 图 4-3 用户登录流程实现图

以下流程图依据当前项目实际代码整理，主要对应：

- `uniapp-project/src/services/auth.js`
- `admin/routes/mp-auth.js`
- `admin/routes/web-auth.js`

## Mermaid 版

```mermaid
flowchart TD
    A["用户进入登录页"] --> B{"登录方式判断"}

    B -->|"微信小程序授权登录"| C["前端调用 uni.login 获取 code"]
    C --> D["前端采集头像、昵称等资料"]
    D --> E["POST /api/mp/auth/login"]
    E --> F["服务端调用 jscode2session"]
    F --> G{"是否成功获取 openid"}
    G -->|"否"| H["返回登录失败信息"]
    G -->|"是"| I["按 openid 查询 users 集合"]
    I --> J{"用户是否存在"}
    J -->|"否"| K["创建新用户记录"]
    J -->|"是"| L["更新昵称、头像等信息"]
    K --> M["生成 JWT Token"]
    L --> M
    M --> N["返回 token 和用户信息"]
    N --> O["前端本地保存 token 与用户信息"]
    O --> P["跳转首页或回跳目标页"]

    B -->|"Web 账号密码登录"| Q["输入账号和密码"]
    Q --> R["POST /api/web/auth/login"]
    R --> S{"账号是否为空"}
    S -->|"是"| H
    S -->|"否"| T["按账号查询 users 集合"]
    T --> U{"用户是否存在"}
    U -->|"否"| V["创建新用户并写入密码摘要"]
    U -->|"是"| W{"是否为后台角色"}
    W -->|"是"| X["拒绝登录，提示走后台入口"]
    W -->|"否"| Y{"账号是否被禁用"}
    Y -->|"是"| Z["返回禁用提示"]
    Y -->|"否"| AA{"是否存在 password_hash"}
    AA -->|"是"| AB["校验密码摘要"]
    AB -->|"失败"| AC["返回账号或密码错误"]
    AB -->|"成功"| AD["补充账号字段并更新时间"]
    AA -->|"否"| AE{"是否为默认密码"}
    AE -->|"否"| AC
    AE -->|"是"| AF["补写密码摘要并更新时间"]
    V --> AG["生成 JWT Token"]
    AD --> AG
    AF --> AG
    AG --> N

    H --> AH["前端提示失败原因"]
    X --> AH
    Z --> AH
    AC --> AH
```

## 论文配图说明

图 4-3 展示了系统用户登录功能的实现流程。系统支持微信小程序授权登录和 Web 账号密码登录两种方式。小程序端先通过 `uni.login` 获取临时登录凭证 `code`，服务端再调用微信接口换取 `openid`，并完成用户查询、注册或信息更新，最后生成 JWT 令牌返回前端。Web 端则通过账号密码调用登录接口，服务端完成账号校验、状态检查、密码摘要验证及令牌签发，登录成功后由前端保存令牌并跳转到目标页面。
