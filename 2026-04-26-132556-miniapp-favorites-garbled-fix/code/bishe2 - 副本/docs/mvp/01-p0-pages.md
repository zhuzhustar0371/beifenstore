# 本地二手交易 MVP 页面清单

## 目标

本次 MVP 只验证一个核心闭环：

`发布商品 -> 浏览商品 -> 发起联系 -> 收到回复`

页面设计必须服务这个闭环，不为完整性增加额外页面。

## 范围原则

- 只做 P0 页面
- 每个页面只保留完成核心任务所需模块
- 能用弹层或组件解决的，不单独拆页面
- 不做支付、物流、信用体系、推荐流、客服系统

## 用户端 P0 页面

### 1. 登录授权页

- 目标：让用户快速完成登录，以便发布商品和发起联系
- 进入方式：首次点击“发布”或“联系卖家”时触发
- 核心模块：
  - 产品一句话说明
  - 微信登录按钮
  - 用户协议与隐私说明
- 必要动作：
  - 一键登录
  - 登录后回跳来源页面
- 验收标准：
  - 用户能在 1 次授权内完成登录
  - 登录后不丢失当前上下文

### 2. 首页商品流

- 目标：让用户快速发现本区县的商品
- 进入方式：默认首页
- 核心模块：
  - 顶部区县选择
  - 搜索框
  - 商品列表
  - 底部导航
- 必要动作：
  - 切换区县
  - 输入关键词搜索
  - 进入商品详情
- 验收标准：
  - 用户能看到已审核商品
  - 区县筛选和关键词搜索有效

### 3. 区县选择弹层

- 目标：在最小范围内完成本地社区定位
- 形态：弹层，不单独做完整页面
- 核心模块：
  - 测试城市列表
  - 区县列表
- 必要动作：
  - 选择当前浏览区县
  - 选择发布商品所属区县
- 验收标准：
  - 区县切换后首页和发布页状态同步

### 4. 商品详情页

- 目标：让用户判断商品是否值得联系
- 核心模块：
  - 商品图片
  - 标题
  - 价格
  - 描述
  - 所在区县
  - 卖家昵称
  - 联系卖家按钮
- 必要动作：
  - 查看商品完整信息
  - 发起联系
- 验收标准：
  - 商品详情信息完整可读
  - 未登录用户点击联系时正确引导登录

### 5. 发布商品页

- 目标：最低成本完成商品发布
- 核心模块：
  - 标题输入
  - 描述输入
  - 价格输入
  - 图片上传
  - 区县选择
  - 提交按钮
- 必要动作：
  - 填写并提交商品
- 验收标准：
  - 用户能成功发布商品
  - 发布后商品状态默认为待审核或待上架

### 6. 消息会话列表页

- 目标：集中查看已发起或收到的联系
- 核心模块：
  - 会话列表
  - 商品缩略信息
  - 最近一条消息
  - 未读标记
- 必要动作：
  - 进入单个会话
- 验收标准：
  - 买家和卖家都能看到与商品相关的会话

### 7. 会话详情页

- 目标：完成买卖双方最小沟通闭环
- 核心模块：
  - 商品卡片
  - 消息记录
  - 文本输入框
  - 发送按钮
- 必要动作：
  - 发送文字消息
  - 查看历史消息
- 验收标准：
  - 双方可在同一会话中持续沟通
  - 消息发送后可被对方看到

### 8. 我的页

- 目标：提供最小账户入口
- 核心模块：
  - 用户基础信息
  - 我的发布入口
  - 我的消息入口
  - 反馈入口
- 必要动作：
  - 进入我的发布
  - 进入消息页
  - 提交反馈
- 验收标准：
  - 页面不承担复杂社交或资产功能

### 9. 我的发布页

- 目标：让发布者查看和管理自己的商品
- 核心模块：
  - 我的商品列表
  - 审核状态
  - 上下架状态
- 必要动作：
  - 查看商品状态
- 验收标准：
  - 用户能知道商品是否已通过审核、是否被下架

### 10. 反馈页

- 目标：为 MVP 收集真实用户反馈
- 核心模块：
  - 问题类型
  - 文本描述
  - 联系方式选填
  - 提交按钮
- 必要动作：
  - 提交反馈
- 验收标准：
  - 反馈能进入后台或数据库

## 管理后台 P0 页面

### 1. 后台登录页

- 目标：管理员进入后台
- 最小能力：
  - 账号登录
- 验收标准：
  - 非管理员无法进入后台

### 2. 商品管理页

- 目标：审核和下架商品
- 核心模块：
  - 商品列表
  - 商品详情侧栏或弹层
  - 审核通过按钮
  - 下架按钮
- 验收标准：
  - 后台能改变商品展示状态

### 3. 用户管理页

- 目标：最低限度处理违规用户
- 核心模块：
  - 用户列表
  - 禁用按钮
  - 状态展示
- 验收标准：
  - 被禁用用户不能继续发布或发消息

### 4. 反馈管理页

- 目标：查看种子用户意见
- 核心模块：
  - 反馈列表
  - 提交时间
  - 处理状态
- 验收标准：
  - 团队能查看用户提交的反馈内容

## 底部导航建议

- 我的

## 明确不做的页面

- 支付页
- 订单页
- 物流页
- 收藏页
- 个人主页
- 客服工作台
- 推荐页
- 活动运营页

## 页面优先级

### 第 1 批必须完成

- 登录授权页
- 首页商品流
- 商品详情页
- 发布商品页
- 会话详情页
- 商品管理页

### 第 2 批可跟进完成

- 消息会话列表页
- 我的页
- 我的发布页
- 反馈页
- 用户管理页
- 反馈管理页
