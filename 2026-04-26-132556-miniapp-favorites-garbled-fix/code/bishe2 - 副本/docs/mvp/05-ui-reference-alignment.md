# UI 参考与 MVP 功能对齐说明

## 目的

这份文档用于把你提供的参考 UI 和当前 MVP 目标对齐，避免后续开发时出现两个问题：

- 只学到了样式，没守住 MVP 范围
- 把参考产品的完整能力误当成第一阶段必须做

当前原则仍然不变：

- 本次只验证 `发布 -> 浏览 -> 联系`
- UI 可以参考闲鱼风格
- 功能不能照搬闲鱼全量能力

## 一、从参考图提取的视觉方向

### 1. 总体风格

- 主色：高识别度亮黄
- 辅助色：黑、白、浅灰
- 强调色：橙红色价格
- 结构特征：
  - 大留白
  - 大圆角
  - 卡片式商品流
  - 顶部强搜索入口
  - 明显的“聊一聊”主按钮

### 2. 可直接借鉴的设计元素

- 顶部黄底导航条
- 圆角搜索框
- 横向分类胶囊标签
- 商品卡片信息层级：
  - 图片
  - 标题
  - 时间/热度
  - 价格
  - 卖家信息
- 详情页左右分栏布局的桌面端思路
- 消息页左会话列表 + 右聊天区布局
- 悬浮快捷入口栏

### 3. 推荐保留的品牌语气

- 轻松、非严肃电商
- 社区感强
- 强调“聊”和“本地”

### 2. 商品详情

### 3. 消息系统

## 三、哪些进入当前 MVP

### P0 必须吸收

- 首页商品流视觉结构
- 顶部搜索
- 商品卡片样式
- 商品详情中的价格、标题、图片区
- `聊一聊` 作为核心主按钮
- 消息页的会话布局
- 反馈入口
- 发布入口

### P0 可弱化实现

- 分类标签
  - 第一阶段可只保留少量固定分类
- 收藏按钮
  - 可先只保留 UI，不一定实现完整功能
- 浏览人数/想要人数
  - 可先只展示浏览量或不展示
- 侧边快捷栏
  - 移动端可转为底部导航 + 我的页入口

### P1/P2 再考虑

- 订单页
- 收货地址管理
- 担保交易
- 立即购买
- 客服系统
- 商家码
- 复杂通知消息
- 收藏系统

## 四、对当前产品的具体落地建议

### 1. 首页

采用：

- 顶部黄底
- Logo + 搜索框
- 下方少量分类标签
- 双列或单列商品流卡片

不采用：

- 过多分类频道
- 各种促销活动入口
- 复杂推荐算法区块

### 2. 商品详情页

建议保留：

- 商品大图
- 价格
- 标题
- 描述
- 区县信息
- 卖家信息
- `聊一聊` 主按钮

建议删除：

- `立即购买`
- 担保交易说明
- 地址与订单明细

原因：

- 这些属于交易闭环，不属于当前验证目标

### 3. 消息页

建议保留：

- 左侧会话列表或移动端顶部 Tab
- 当前商品摘要
- 文本聊天输入框

建议删除：

- 交易状态链路
- 退款/售后提示
- 平台客服强引导

### 4. 发布入口

参考图中的“发闲置”入口是有价值的。  
MVP 建议：

- 底部导航直接放 `发布`
- 或首页右下角放悬浮发布按钮

### 5. 反馈入口

参考图里的“反馈”入口值得保留。  
这和 MVP 的“快速收集种子用户反馈”高度一致。

## 五、推荐的视觉规范

### 颜色

- 主色：`#FFE000` 附近的亮黄
- 文字主色：`#222222`
- 次级文字：`#888888`
- 页面背景：`#F6F6F6`
- 卡片背景：`#FFFFFF`
- 价格色：`#FF5A1F`
- 聊天主按钮：黄色或黄黑组合

### 组件

- 搜索框：超大圆角
- 按钮：圆角胶囊
- 卡片：12px 到 20px 圆角
- 标签：浅灰底圆角胶囊

### 字体层级

- 标题：中粗
- 价格：高对比、大字号
- 辅助信息：小号灰字

## 六、桌面端与移动端策略

### 移动端

优先实现：

- 首页
- 详情页
- 发布页
- 消息页
- 我的页

### 桌面端

如果后续同时做 Web，可参考你给的桌面截图：

- 顶部横向导航
- 列表页更宽
- 详情页左右分栏
- 消息页左列表右聊天

但当前 MVP 首发仍建议以移动端优先。

## 七、主脑结论

这组参考图非常有价值，但它提供的是：

- `风格参考`
- `成熟平台信息架构参考`
- `聊天与详情页布局参考`

不是第一阶段功能清单。

当前应采取的策略是：

- UI 风格尽量向参考图靠拢
- 功能范围严格收缩到 MVP
- 在详情页保留 `聊一聊`
- 暂时砍掉 `立即购买 / 订单 / 地址 / 担保交易`

## 八、建议交给其他 AI 的执行口径

你后面如果把前端实现交给其他 AI，可以直接给它这条执行口径：

> 参考闲鱼风格做移动端 UI，保留黄黑白主视觉、卡片流、搜索、商品详情、消息聊天等布局语言；但功能只实现 MVP 的发布、浏览、联系和反馈，不实现订单、支付、地址、担保交易、客服系统。
