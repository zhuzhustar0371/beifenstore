# 2026-04-20 全局设计系统改造记录

- 记录时间：2026-04-20 15:38

- 变更文件：
  - `zhixi-website/frontend/src/styles.css`
  - `zhixi-website/frontend/src/App.vue`

## 本阶段目标

- 把零散样式改成可复用的全局设计系统。
- 收敛字体、字重、对比度、边框、圆角、按钮和卡片规则。
- 优化站点头部与底部的品牌层级和导航一致性。

## 具体改动

### 1. 重写全局设计令牌

- 统一主色为更克制的绿色体系，辅色为有目的的琥珀色，用于价格和重点动作。
- 提高边框和说明文字对比度，避免之前边框过浅、说明字太淡的问题。
- 保留非纯黑文本策略，正文统一使用深灰蓝而不是纯黑。

### 2. 收敛排版规范

- 字体栈统一为中文界面友好的无衬线字体：`Noto Sans SC / PingFang SC / Microsoft YaHei / sans-serif`。
- 默认正文行高统一为 1.6，并在说明文字、段落、列表中提升到更接近 1.7~1.8。
- 全局上只保留“常规”和“粗体”两档思路，移除原先大量 500 / 600 / 800 的混用方向。

### 3. 建立一致的组件基础样式

- 统一按钮高度、圆角、边框、hover/focus 规则。
- 统一卡片、统计块、规则卡、数据项、标签的边框与留白。
- 新增页面头部、分区标题、弹窗区块、空状态、列表说明等公共类。

### 4. 优化壳层结构

- 头部品牌区改为“品牌字标 + 副标题”双层结构，层级更清晰。
- 登录态用户名改为轻量胶囊展示，和按钮风格统一。
- 页脚文案改得更完整，导航与品牌说明分组更清晰。
- 移动端导航保留折叠方式，但面板和按钮外观统一到同一设计语言。

## 为什么这样改

- 你要求“保持一致性”“看起来相似的元素在功能上也类似”，所以先把所有通用元素收束到一套基础系统里。
- 你要求“确保元素 3:1 对比度、文本 4.5:1 对比度”，所以本阶段优先处理颜色和边框基础值。
- 你要求“删减不必要的样式”，所以去掉了很多偏装饰性的散乱设定，把视觉重点集中到结构、留白和清晰分组上。

## 当前结果

- 后续页面和弹窗都可以直接复用新的页面头、卡片、分组、按钮、表单规则。
- 本阶段已完成，下一阶段开始改首页、规则页、用户中心的实际结构和内容分组。
