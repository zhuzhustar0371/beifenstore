# 2026-04-20 弹窗与反馈层优化记录

- 记录时间：2026-04-20 15:54
- 操作阶段：登录弹窗、下单弹窗、反馈提示统一
- 变更文件：
  - `zhixi-website/frontend/src/components/LoginModal.vue`
  - `zhixi-website/frontend/src/components/OrderModal.vue`
  - `zhixi-website/frontend/src/components/FeedbackAlert.vue`
  - `zhixi-website/frontend/src/styles.css`

## 本阶段目标

- 把交互层也统一到新的视觉系统里。
- 让弹窗不只是“能用”，而是有清晰分组、稳定层级和一致语义。
- 修复“只靠颜色表达状态”的问题。

## 登录弹窗改动

### 1. 头部层级更清晰

- 新增“账户入口”小标题和明确的主标题。
- 登录、注册、重置密码三种模式现在都有更一致的入口说明。

### 2. 表单按模块分组

- 所有输入字段都放进统一的 `modal-section` 中。
- 这样用户更容易理解“这是一个完整表单块”，而不是散乱的输入项。

### 3. 模式说明更明确

- 使用 `modeDescription` 统一输出模式说明文案。
- 登录、注册、重置密码都能在切换后立即看到当前动作的目的。

### 4. 交互细节增强

- 给手机号、验证码、密码字段补上更合适的 `autocomplete` 和 `inputmode`。
- 所有功能按钮补齐 `type="button"`，避免潜在默认提交行为。
- 收起了多余的临时样式，让输入区和切换区保持一致。

## 下单弹窗改动

### 1. 订单确认分成两组

- 第一组是商品与数量，第二组是收货信息。
- 比之前直接从商品卡跳到表单更清晰，用户更容易确认“我现在正在填写哪一部分”。

### 2. 补充解释性文本

- 在数量区域增加说明，在收货信息前增加说明，在提交按钮前增加信任提示。
- 这让弹窗不只负责输入，也负责解释后续流程。

### 3. 去掉内联样式

- 原有 `max-width` 和局部 spacing 的内联样式被收回到统一类名里。
- 方便今后复用、维护和回滚。

## 反馈提示改动

### 1. 不再只靠颜色区分状态

- 增加图标和默认标题，例如“提示 / 操作成功 / 请注意 / 操作失败”。
- 即使用户对颜色不敏感，也能通过图标与文字理解当前状态。

### 2. 提升语义表达

- 增加 `role="status"` 和 `aria-live="polite"`。
- 让状态变化在辅助技术场景下也更容易被感知。

### 3. 对齐全局系统

- 使用新的全局颜色变量、边框和圆角值。
- 让反馈提示和页面卡片、表单、弹窗属于同一套设计系统，而不是孤立的小组件。

## 当前结果

- 交互入口、下单流程和反馈提示都已经从“零散组件”变成“同一语言下的统一模块”。
- 后续如果构建成功，这一轮 UI 优化就已经具备可发布和可回退的基础条件。
