## 现状判断

### 1. 视觉层级还不够稳定

- 首页虽然有 Hero、商品、返现、流程分区，但多个区块仍然依赖居中标题和相似卡片，信息主次感不够明显。
- 规则页和用户中心页大量依赖默认卡片和表格，没有形成明确的“页面头部 -> 关键摘要 -> 详细内容”层级。
- 弹窗中的订单信息、表单信息、反馈提示虽然都存在，但分组边界较弱，阅读路径不够顺。

### 2. 留白和分组有基础，但不统一

- 多个页面存在内联样式，说明页面级 spacing 规则没有完全沉淀为统一类名。
- 卡片、列表、表格、统计块之间的间距并不完全一致，相似组件没有共享统一的外观语言。
- 用户中心中的邀请码、空状态、刷新按钮等信息使用了临时样式，分组方式偏“补丁式”。

### 3. 一致性不足

- 全局样式中存在大量不同字重：400、500、600、700、800 混用，和“只使用正常与粗体”原则不一致。
- 存在大写与较强字距使用，例如产品兜底视觉中的 `ZHIXI`、多处 `letter-spacing`，会削弱亲和度和可读性。
- 不同卡片看起来相似，但内部用途不完全一致，例如规则卡、数据卡、统计卡、返现卡的标题结构和底部说明方式不统一。

### 4. 可访问性需要加强

- 现有 `--color-text-muted: var(--gray-400)` 在白底上对比度偏低，不适合作为正文或重要说明文本。
- `--color-border: var(--gray-200)` 在白底下过浅，不利于满足 3:1 的界面元素分隔对比。
- `FeedbackAlert` 主要依靠颜色区分状态，缺少更明确的图标/文本语义提示。

### 5. 样式可以继续删减

- 页面里有多处 `style=""` 内联样式，可回收到全局或局部类中。
- 一些装饰性强但信息价值有限的设置可以收敛，避免界面显得“每块都很重”。
- 首页局部背景色写在模板内，不利于后续复用和统一维护。

## 本次优化方向

### A. 先统一设计令牌

- 统一字体栈为中文界面友好的无衬线字体。
- 收敛字重，只保留 400 和 700 两档。
- 提升文本、边框、标签、表格等元素的基础对比度。
- 统一卡片圆角、阴影、段落行高、分区留白和焦点态。

### B. 重做页面层级与分组

- 首页改为更明确的“价值主张 -> 核心卖点 -> 商品区 -> 返现说明 -> 流程”的阅读顺序。
- 规则页增加摘要区与说明区，弱化“纯表格堆叠”的感觉。
- 用户中心强化账户概览、邀请码、统计、记录三层结构。

### C. 统一组件语言

- 让按钮、标签、卡片头、列表项、表单区块遵循相同间距、字重和状态规则。
- 登录弹窗和下单弹窗都补强分组标题、说明文字和反馈区样式。
- 反馈提示加入图标和标题，避免只用颜色传达状态。

## 风险与控制

- 风险：`styles.css` 体量较大，直接调整全局令牌可能影响多页面联动样式。
- 控制方式：先做源码备份；每个修改阶段单独记录；最终执行构建验证；保留清晰回滚路径。
