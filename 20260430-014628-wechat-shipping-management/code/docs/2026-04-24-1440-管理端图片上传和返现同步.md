# 管理端图片上传 + 多端返现金额同步

## 日期
2026-04-24

## 需求
1. 管理端商品编辑支持上传图片
2. 上传后小程序、官网同步显示同一图片
3. 官网首页返现金额与后端实时同步

## 现状

### 已有基础设施（不需改动）
| 链路 | 状态 |
|------|------|
| 后端图片上传 API | `POST /api/admin/uploads/image` ✅ |
| 图片公开访问 | `GET /api/uploads/{filename}` ✅ |
| 商品 `imageUrl` 入库 | `AdminProductUpsertRequest` 已有 `imageUrl` 字段 ✅ |
| 多端读取 | 小程序/官网均通过 `GET /api/products` 获取 ✅ |
| 返现规则实时计算 | `GET /api/cashbacks/rules` 实时算 ✅ |

### 需要改的
| 问题 | 文件 |
|------|------|
| 管理端编辑弹窗没有图片上传 | `ProductsPage.vue` |
| 官网首页返现金额写死 | `HomePage.vue` |
| 小程序首页返现金额可能写死 | 需检查 `wechat-app` |

## 改动方案

### 1. ProductsPage.vue — 编辑弹窗增加图片上传
- 弹窗中增加「商品图片」区域：当前图片预览 + 上传按钮
- 调用 `POST /api/admin/uploads/image` 上传
- 上传成功后将 URL 写入 `editForm.imageUrl`
- 保存时通过 `updateProduct` 提交

### 2. Website HomePage.vue — 返现金额动态获取
- 移除硬编码的 `personalCashbacks` 和 `inviteCashbacks`
- 改为调用 `fetchCashbackRules()` API 获取实时数据
