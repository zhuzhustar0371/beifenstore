- 用户反馈：开发者工具里可登录，真机调试里不行，界面显示 `request ok`

  - 当前小程序登录页已被改造成三步登录流程：登录、上传头像、填写昵称
  - 当前前端并未按“是否新用户”分流，而是按“头像和昵称是否完整”分流

## 实际修改

### 1. 后端返回新用户标志位

- 文件：`G:\zhiximini\backend-api\src\main\java\com\zhixi\backend\service\UserAuthService.java`
- 修改内容：
  - 在小程序登录入口中新增 `isFirstMiniappLogin(...)` 判断
  - 在 `/api/auth/wechat-miniapp/login` 返回体中追加：
    - `isNewUser`
    - `needProfileCompletion`
- 判断原则：
  - 若当前 `openid`、`unionid`、小程序授权记录均不存在，则判定为首次创建账号
  - 若已有关联账号或历史授权记录，则判定为非新用户

### 2. 小程序登录页按新老用户分流

- 文件：`G:\zhiximini\wechat-app\pages\login\login.js`
- 修改内容：
  - 新增本地状态键：`miniappProfileCompletionPending`
  - 登录成功后：
    - 非新用户：直接提示登录成功并进入 `/pages/user/user`
    - 新用户：进入头像上传和昵称填写流程
  - `onShow()` 逻辑调整为：
    - 若不存在待补资料标记，则直接进入程序
    - 若存在待补资料标记，则继续走步骤 2/3
  - 新用户在资料补全成功后会清除待补资料标记
  - 新用户在资料保存失败但本地兜底进入时，也会清除待补资料标记，避免再次被卡住
