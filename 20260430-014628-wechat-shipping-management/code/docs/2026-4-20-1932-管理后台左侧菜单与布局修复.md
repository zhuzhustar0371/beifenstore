# 管理后台左侧菜单与布局修复

## 记录时间

- 时间：2026-04-20 19:32（Asia/Shanghai）
- 操作人：Codex
- 目标：修复管理后台卡片挤压、布局混乱的问题，并恢复左侧菜单栏布局。

## 用户反馈

用户反馈：

- “恢复只恰当呢布局，布局乱了”
- 随后明确：
  - “菜单栏在左列”

截图中可见问题：

- 用户、商品、订单等模块挤在同一行
- 订单管理中物流输入框和发货按钮被挤压
- 页面缺少左侧菜单栏
- 长表格不适合放在窄列卡片内

## 本次修改

### 1. 重整管理后台页面结构

修改文件：

- `zhixi-website/admin-frontend/src/App.vue`

页面结构调整为：

- 顶部：后台标题、欢迎信息、刷新、退出登录
- 左侧：固定菜单栏
- 右侧：主要内容区

左侧菜单包含：

- 看板概览
- 用户管理
- 商品管理
- 订单管理
- 邀请管理
- 返现流水

### 2. 调整内容区布局

右侧内容区规则：

- 顶部指标卡保持一行三列
- 用户、商品、邀请作为紧凑卡片
- 订单管理独占整行，用表格展示
- 返现流水独占整行，用表格展示

这样避免：

- 物流单号输入框被挤压
- 发货按钮过窄
- 批准打款按钮挤在窄列中
- 返现表格横向撑破页面

### 3. 保留已有功能

本次保留：

- 商品上下架
- 订单发货
- 返现流水中文显示
- `PENDING` 返现的“批准打款”按钮
- 调用 `POST /api/admin/cashbacks/{cashbackId}/transfer`
- 枚举中文映射

### 4. CSS 布局修复

修改文件：

- `zhixi-website/admin-frontend/src/styles.css`

新增/调整：

- `.admin-body`：左侧菜单 + 右侧内容的两栏布局
- `.sidebar`：左侧菜单，桌面端 sticky
- `.dashboard-grid`：右侧内容网格
- `.wide-card`：订单与返现模块跨满右侧整行
- 移动端下左侧菜单自动变为顶部双列按钮
