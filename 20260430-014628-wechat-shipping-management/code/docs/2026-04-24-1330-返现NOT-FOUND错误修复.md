# 返现同步状态 NOT_FOUND 错误修复

## 日期
2026-04-24

## 问题
批准返现后，点击「同步状态」按钮报错：
```
微信商家转账状态查询失败：NOT_FOUND - 记录不存在
```

## 根因分析

### 流程还原

`transferCashback()` 方法执行顺序：

1. **`markTransferProcessing()`** — 先将本地生成的 `outBatchNo`（`ZXCB{id}`）和 `outDetailNo`（`ZXCBD{id}`）写入数据库，状态改为 `PROCESSING`
2. **`transferToWallet()`** — 调微信 API 创建转账。若微信拒绝（余额不足、无权限等），抛出 `BusinessException`
3. **catch 块** — 将状态改为 `FAILED`，**但把本地生成的 `outBatchNo`/`outDetailNo` 也写入了数据库**

```java
// catch 块中，outBatchNo/outDetailNo 是本地生成的，微信那边根本不存在
cashbackRecordMapper.updateTransferResult(
    cashbackId,
    failedStatus,       // "FAILED"
    cb.getOutBatchNo(), // ← 微信不存在
    cb.getOutDetailNo(),// ← 微信不存在
    ...
);
```

### 触发路径

数据库中有 `outBatchNo` → 前端渲染出 **「再次同步」** 按钮 → 用户点击 → 调用 `queryTransferDetailByOutNo()` 拿着本地单号问微信 → 微信返回 **NOT_FOUND**（该单号从未在微信端注册过）

### 官方文档依据

根据 [微信支付商家转账API文档](https://pay.weixin.qq.com/docs/merchant/apis/transfer/transfer-bill/query-transfer-bill-by-out-no.html)：

> **NOT_FOUND（HTTP 404）**：查询的转账明细单不存在。
> 可能原因：转账受理失败，明细单未落地；或明细单号不属于该商户。

## 修复方案

### 修改文件

| 文件 | 改动 |
|------|------|
| `CashbackRecordMapper.java` | 新增 `resetTransferForRetry()` 方法 |
| `AdminManageService.java` | `syncCashbackTransfer()` 捕获 NOT_FOUND 时自动重置 |

### 修复逻辑

在 `syncCashbackTransfer()` 中，当微信返回 NOT_FOUND 时判断：

- 若当前记录状态是 **FAILED** → 说明转账从未创建成功。重置 `outBatchNo`/`outDetailNo`/`status` 为 `PENDING`，让用户可以重新点击「批准打款」重试
- 若当前记录状态是 **PROCESSING/TRANSFERING** → 可能是微信尚未处理完成，仅返回错误信息，不改动数据

### 影响范围

- 仅影响 `syncCashbackTransfer` 的异常处理分支
- 正常查询到结果的流程不受影响
- 仅对 `status=FAILED` + `NOT_FOUND` 的组合触发重置
