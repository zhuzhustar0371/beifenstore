# 商户信息页面优化计划（终版）

## 原则
- **所有商户数据通过服务器环境变量注入**，不出现在任何源码文件中
- 源码中 `application.yml` 仅添加**占位符引用** `${VAR_NAME:}`，默认值为空

## 实施步骤

### Step 1: 后端修改（3 个文件）

#### 1.1 application.yml — 新增占位符配置（无实际值）
```yaml
app:
  merchant:
    info:
      apply-sn: ${MERCHANT_APPLY_SN:}
      short-name: ${MERCHANT_SHORT_NAME:}
      full-name: ${MERCHANT_FULL_NAME:}
      miniapp-nickname: ${MERCHANT_MINIAPP_NICKNAME:}
      miniapp-type: ${MERCHANT_MINIAPP_TYPE:}
      miniapp-subject: ${MERCHANT_MINIAPP_SUBJECT:}
```

已有字段（直接复用）：
- `mchId` ← `WECHAT_PAY_MCHID`
- `appId` ← `WECHAT_PAY_APPID`（小程序 AppID）
- `merchantSerialNumber` ← `WECHAT_PAY_SERIAL_NUMBER`（证书序列号）
- `publicKeyId` ← `WECHAT_PAY_PUBLIC_KEY_ID`（公钥ID）
- `apiV3Key` ← `WECHAT_PAY_API_V3_KEY`
- `webAppId` ← `WECHAT_WEB_APP_ID`
- `webAppSecret` ← `WECHAT_WEB_APP_SECRET`

#### 1.2 WechatPayService.java — 新增 `getMerchantInfo()` 方法
注入新增字段，复用已有 `@Value` 字段，组装 Map 返回。

#### 1.3 AdminController.java — 新增 GET 端点
```
GET /api/admin/merchant/info → ApiResponse<Map<String, String>>
```

### Step 2: 前端修改（4 个文件）

#### 2.1 新建 `MerchantPage.vue`
- 5 张玻璃拟态信息卡片（商户/小程序/证书/API密钥/公众号）
- 敏感字段掩码 + 显隐切换 + 一键复制
- 加载骨架屏
- 遵循现有 Tailwind 设计体系

#### 2.2 `router.js` — 添加 `/merchant` 路由
#### 2.3 `MainLayout.vue` — 侧边栏添加"商户信息"菜单项
#### 2.4 `api.js` — 添加 `fetchMerchantInfo()`

### Step 3: 部署时在服务器 .env 补充环境变量
```bash
MERCHANT_APPLY_SN=201699297494
MERCHANT_SHORT_NAME=知禧净享生活
MERCHANT_FULL_NAME=石家庄知禧健康管理有限公司
MERCHANT_MINIAPP_NICKNAME=知禧净享生活
MERCHANT_MINIAPP_TYPE=小程序
MERCHANT_MINIAPP_SUBJECT=石家庄知禧健康管理有限公司
```
