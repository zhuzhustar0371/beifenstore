## 1. 需求与问题确认

用户反馈:
1. 现有登录后“我的”页未展示真实头像昵称，显示默认头像/默认昵称。
2. 目标流程为:
   - 步骤1: 点击登录，获取 token
   - 步骤2: 登录按钮隐藏，显示上传头像按钮
   - 步骤3: 上传头像后，显示昵称输入与“进入程序”按钮
   - 步骤4: 提交昵称+头像到服务端
   - 步骤5: 在“我的”页展示真实头像和昵称

排查结论:
- 前端登录页已接近分步流程，但仍使用 `open-type=getUserInfo` 作为第一步触发，和“仅取 token”目标不一致。
- “我的”页未对头像 URL 做绝对地址归一化，且已登录但资料不完整时不会强制回到补全流程。
- 后端 `UserAuthService.java` 存在多处字符串断裂，`mvn compile` 失败，影响发布。

## 3. 代码修改清单

### 3.1 小程序前端

1) `wechat-app/pages/login/login.wxml`
- 第一阶段登录按钮改为普通点击触发:
  - 从 `open-type="getUserInfo" + bindgetuserinfo` 改为 `bindtap="onGetUserInfo"`
- 保留三阶段 UI:
  - `loginStep=1` 微信授权一键登录
  - `loginStep=2` `chooseAvatar`
  - `loginStep=3` 昵称输入 + 进入程序

2) `wechat-app/pages/login/login.js`
- 新增 `onShow`：若已有 token，根据资料完整度直接进入第2/3步或直接进入“我的”页。
- `onGetUserInfo` 改为“只做 token 登录 + 状态机推进”，不依赖 `getUserInfo` 授权结果。
- 上传头像与资料提交后，统一写入 `wx.setStorageSync('userInfo', ...)`。
- 新增 `toAbsoluteUrl()`：头像 URL 统一绝对化，避免相对地址导致图片不展示。
- 新增 `isDefaultWechatNickname()`：识别默认昵称（如 `微信用户xxxx`），用于判断资料是否完成。

3) `wechat-app/pages/user/user.js`
- 重建页面逻辑，保持原业务入口不变，新增以下能力:
  - `checkLogin()` 先读取缓存并归一化头像地址
  - `refreshUserProfile()` 拉取 `/api/auth/me` 并回写缓存
  - `ensureProfileCompleted()` 若头像或昵称未完成，自动跳转登录页补全
  - `requireLogin()` 统一未登录拦截提示
- 保留收益统计、菜单跳转、退出登录行为。

4) `wechat-app/pages/user/user.wxml`
- 重建为规范结构:
  - 优先显示 `userInfo.avatarUrl`
  - 头像缺失时回退 `open-data userAvatarUrl`
  - 昵称优先显示 `userInfo.nickname`
  - 缺失时回退 `open-data userNickName`
- 修复文案与标签结构，确保可正确渲染。
