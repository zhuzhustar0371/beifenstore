# 我的订单商品图片联动发布

## 1. 目标

让小程序「购买记录」和「订单详情」页面的商品图，不再显示固定的红色占位块，而是根据订单里的 `productId` 联动显示对应商品的主图，同时把商品名称一并带出来，避免订单页和商品页信息不一致。

## 2. 现状分析

### 2.1 前端现状

- 订单列表页当前固定写死了占位图和商品名。
- 订单详情页同样固定写死了占位图和商品名。
- 页面本身已经能拿到订单 `id`、`productId`、`quantity`、`totalAmount` 等字段，但没有直接可渲染的商品图字段。

### 2.2 后端现状

- `GET /api/orders/user/{userId}` 和 `GET /api/orders/{orderId}` 当前返回的是 `Order` 模型。
- `Order` 模型里只有 `productId`，没有 `productName`、`productImageUrl`。
- `OrderMapper` 的查询只查了订单表，没有把 `products` 表里的商品名称和主图一起带出来。

### 2.3 处理思路

- 后端在订单查询里 `LEFT JOIN products`，直接返回商品名称和主图地址。
- 小程序侧统一做一次订单归一化，负责：
  - 商品名称兜底
  - 商品主图地址转换
  - 占位图兜底
- 列表页和详情页统一读取新字段，避免两套逻辑各自处理。
