# 返现待结算状态修复

## 记录时间

- 时间：2026-04-20 18:38（Asia/Shanghai）
- 操作人：Codex
- 目标：修复小程序返现页顶部待结算金额显示为 `0.00`、明细状态显示“已取消”的问题。

## 用户反馈

用户在小程序返现记录页截图中反馈：

- 明细中已经出现 `+0.01`
- 顶部“待结算佣金”仍显示 `0.00`
- 明细状态显示“已取消”

## 问题定位

### 1. 返现金额已经生成

截图显示明细中存在：

- 自购返现：`+0.01`

因此不是返现没有生成，而是状态和统计口径问题。

### 2. 小程序端状态映射不完整

文件：

- `wechat-app/pages/cashback/cashback.js`

旧逻辑：

- 只有 `PENDING` 会计入顶部待结算金额
- `TRANSFERRED` 显示为已到账
- 其它任何状态都会显示为已取消

问题：

- 后端生成的返现状态曾是 `SETTLED`
- 小程序不认识 `SETTLED`
- 所以 `SETTLED` 没被计入待结算，并被显示成“已取消”

### 3. 后端返现状态与提现接口不一致

后端返现生成逻辑：

- `backend-api/src/main/java/com/zhixi/backend/service/CashbackService.java`

后台提现接口逻辑：

- `backend-api/src/main/java/com/zhixi/backend/service/AdminManageService.java`

发现：

- 新返现生成时写入 `SETTLED`
- 但后台打款接口只允许 `PENDING` 状态发起转账

结论：

- 新返现记录应该写为 `PENDING`
- `PENDING` 才符合“待结算/可申请提现/可后台转账”的业务口径

## 本次代码修改

### 1. 后端新返现状态改为 PENDING

修改文件：

- `backend-api/src/main/java/com/zhixi/backend/service/CashbackService.java`

修改内容：

- 个人返现新记录状态：`PENDING`
- 邀请返现新记录状态：`PENDING`

影响：

- 后续新增返现会自动进入待结算状态
- 后台打款接口可识别并处理

### 2. 小程序返现页状态映射修复

修改文件：

- `wechat-app/pages/cashback/cashback.js`

修改内容：

- 支持后端字段 `type`
- 兼容旧字段 `cashbackType`
- `PENDING` 与 `SETTLED` 都计入顶部待结算
- 状态文案：
  - `TRANSFERRED`：已到账
  - `PROCESSING`：处理中
  - `CANCELLED`：已取消
  - 其它状态默认：待结算
- 类型文案：
  - 邀请相关：邀请返现
  - 其它：自购返现

说明：

- 兼容 `SETTLED` 是为了让历史记录即使未迁移也能在小程序端显示为待结算
- 后端已改为新记录写 `PENDING`
