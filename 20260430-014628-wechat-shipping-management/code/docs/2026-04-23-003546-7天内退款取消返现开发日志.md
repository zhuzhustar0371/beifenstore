## 2. 开发前分析

- 当前系统在订单支付成功后，会生成自购返现，并在被邀请用户首单支付时触发邀请批次返现。
- 当前退款链路只处理订单退款状态，不会同步取消返现，也不会回退邀请首单标记。
- 当前前端返现统计存在口径问题：
  - 小程序返现页会把所有返现记录金额直接累加
  - Web 用户中心返现累计也会把所有返现记录金额直接累加
  - 若仅将返现状态改为 `CANCELLED`，页面总额仍会继续计入，需同步修正统计口径

## 3. 本次实现目标

- 新增退款申请时间记录，便于判断是否属于“支付后 7 天内退款”
- 退款成功后，若命中 7 天规则：
  - 取消该订单对应自购返现
  - 若该订单是被邀请人首单，则回退首单标记
  - 如因此导致邀请批次返现不再成立，则取消对应待结算邀请返现
- 小程序和 Web 用户中心返现累计排除已取消返现
- 后台总返现统计排除已取消返现
