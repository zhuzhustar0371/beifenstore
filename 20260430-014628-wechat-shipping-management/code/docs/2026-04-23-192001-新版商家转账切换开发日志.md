# 2026-04-23 新版商家转账切换开发与发布日志

## 1. 基本信息

- 执行时间：2026-04-23 19:20 - 19:38（Asia/Shanghai）
- 操作人：Codex
- 用户批准指令：`批准`
- 触发问题：管理后台返现批准打款时报错 `NO_AUTH - 当前商户号接入升级版本功能，暂不支持使用升级前功能`。
- 目标：将返现打款从旧版 `/v3/transfer/batches` 切换到新版商家转账 `/v3/fund-app/mch-transfer/transfer-bills`，避免 `NO_AUTH`，并继续保留状态同步、失败原因和回退能力。

## 2. 官方文档依据

- 新版普通商户发起转账接口：`POST /v3/fund-app/mch-transfer/transfer-bills`
  - 文档：`https://pay.weixin.qq.com/doc/v3/merchant/4012716434.md`
- 新版普通商户按商户单号查单接口：`GET /v3/fund-app/mch-transfer/transfer-bills/out-bill-no/{out_bill_no}`
  - 文档：`https://pay.weixin.qq.com/doc/v3/merchant/4012716437.md`
- 现金营销场景参数：`transfer_scene_id=1000`，`transfer_scene_report_infos` 必须包含 `活动名称`、`奖励说明`
  - 文档：`https://pay.weixin.qq.com/doc/v3/merchant/4013774588.md`
- 关键结论：新版用户确认模式下，HTTP 200 且 `state=WAIT_USER_CONFIRM` 时代表创建成功，需要用返回的 `package_info` 引导用户确认收款。

## 4. 本次修改范围

### 4.1 backend-api

- `src/main/java/com/zhixi/backend/model/CashbackRecord.java`
  - 新增 `transferPackageInfo` 字段，用于保存新版接口返回的用户确认参数。

- `src/main/resources/schema.sql`
  - `cashback_records` 新增 `transfer_package_info VARCHAR(1024) NULL`。

- `src/main/java/com/zhixi/backend/config/DatabaseMigrationRunner.java`
  - 增加 `cashback_records.transfer_package_info` 启动迁移。

- `src/main/java/com/zhixi/backend/mapper/CashbackRecordMapper.java`
  - 查询结果补充 `transfer_package_info AS transferPackageInfo`。
  - `markTransferProcessing` 支持传入当前状态，允许旧版 `NO_AUTH` 失败单重新发起新版转账。
  - `updateTransferResult` 增加 `transferPackageInfo` 落库。

- `src/main/java/com/zhixi/backend/service/WechatPayService.java`
  - 移除返现打款对旧 SDK `TransferBatchService.initiateBatchTransfer` 的依赖。
  - 使用 SDK 通用 `HttpClient` 调用新版接口：
    - 发起：`/v3/fund-app/mch-transfer/transfer-bills`
    - 查单：`/v3/fund-app/mch-transfer/transfer-bills/out-bill-no/{out_bill_no}`
  - 请求参数：
    - `transfer_scene_id=1000`
    - `user_recv_perception=现金奖励`
    - `transfer_scene_report_infos`: `活动名称=知禧返现`、`奖励说明=消费返现奖励`
  - 使用 `out_detail_no` 作为新版 `out_bill_no`，避免更换单号造成重复转账风险。
  - 保存新版返回的 `transfer_bill_no`、`state`、`package_info`、`fail_reason`。

- `src/main/java/com/zhixi/backend/service/AdminManageService.java`
  - `NO_AUTH` 旧失败单可通过后台“新版重试”重新发起。
  - 状态映射：
    - `WAIT_USER_CONFIRM` -> `WAIT_USER_CONFIRM`
    - `TRANSFERING` -> `TRANSFERING`
    - `SUCCESS` -> `TRANSFERRED`
    - `FAIL` -> `FAILED`
    - `CANCELLED` -> `CANCELLED`
  - 网络异常保留为 `PROCESSING`，便于按原单号同步状态，避免误换单。

- `src/main/resources/application.yml`
  - 新增新版转账可配置项：
    - `WECHAT_PAY_TRANSFER_NOTIFY_URL`
    - `WECHAT_PAY_TRANSFER_SCENE_ID`
    - `WECHAT_PAY_TRANSFER_ACTIVITY_NAME`
    - `WECHAT_PAY_TRANSFER_REWARD_DESC`
    - `WECHAT_PAY_TRANSFER_USER_RECV_PERCEPTION`

### 4.2 zhixi-website/admin-frontend

- `src/views/CashbacksPage.vue`
  - 状态筛选新增：`待用户确认`、`转账中`。
