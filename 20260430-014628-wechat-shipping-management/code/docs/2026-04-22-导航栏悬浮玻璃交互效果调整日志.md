# 导航栏悬浮玻璃交互效果调整日志

## 1. 目标

在当前已经完成的半透明磨砂导航基础上，继续增加“鼠标移动或手指触摸时的悬浮玻璃反馈”。

本次目标分两部分：

- 顶部自定义导航栏：触摸/悬停时有更明显的浮起、增亮和模糊增强效果。
- 底部 custom tabBar：每个 tab 在触摸/悬停时有玻璃泡泡上浮效果，和顶部保持统一视觉语言。

## 2. 现状分析

### 2.1 顶部导航栏

- `pages/user/user.json` 已经改成 `navigationStyle: "custom"`。
- `pages/user/user.wxml` 顶部导航已经是自定义玻璃栏，而不是系统导航栏。
- `pages/user/user.wxss` 已经具备半透明背景、边框、阴影和 `backdrop-filter`。

### 2.2 底部导航栏

- 项目使用的是自定义 `custom-tab-bar`。
- `custom-tab-bar/index.wxss` 已经有半透明卡片、模糊、高光和圆形 bubble。
- 但在触摸/悬停时，还缺少更明确的“悬浮反馈”。

### 2.3 交互限制

- 小程序里没有网页那种完整的 CSS `:hover` 体系可以直接照搬。
- 实现方式要依赖小程序原生支持的 `hover-class`、`hover-start-time`、`hover-stay-time`。
- 所以本次方案是：
  - 触摸态优先
  - 桌面开发者工具/鼠标悬停尽量复用同一套 hover 样式

## 4. 实际修改

### 4.1 顶部导航栏悬浮态

修改文件：

- `[wechat-app/pages/user/user.wxml](/G:/zhiximini/wechat-app/pages/user/user.wxml)`
- `[wechat-app/pages/user/user.wxss](/G:/zhiximini/wechat-app/pages/user/user.wxss)`

修改内容：

- 给顶部 `user-nav` 增加 `hover-class="user-nav-hover"`。
- 设置较短的 `hover-start-time` 和 `hover-stay-time`，让触摸反馈更快。
- 增加 `user-nav-hover` 的视觉样式：
  - 背景更亮一点
  - 模糊半径更强一点
  - 阴影更深一点
  - 整体轻微上浮
- 给标题和高光层增加过渡动画，避免状态切换生硬。

效果预期：

- 鼠标移入或手指按下时，顶部导航像一块轻微抬起的玻璃卡片。
- 松开后回到基础磨砂状态。

### 4.2 底部 tabBar 悬浮态

修改文件：

- `[wechat-app/custom-tab-bar/index.wxml](/G:/zhiximini/wechat-app/custom-tab-bar/index.wxml)`
- `[wechat-app/custom-tab-bar/index.wxss](/G:/zhiximini/wechat-app/custom-tab-bar/index.wxss)`

修改内容：

- 给每个 `tab-item` 增加 `hover-class="tab-item-hover"`。
- 让 tab 项在触摸/悬停时统一进入浮起态。
- 增加 hover 时的视觉变化：
  - bubble 上浮
  - bubble 变亮
  - 阴影增强
  - label 字体颜色变深
- 保留 active 状态与 hover 状态同时存在时的视觉一致性。

效果预期：

- 手指按住某个 tab 时，tab bubble 会像一颗浮起来的玻璃珠。
- 在桌面端开发者工具里也会表现出更明显的悬停感。

### 4.3 说明

- 本次没有新增 JS 逻辑文件。
- 交互主要依赖小程序原生 `hover-class` 能力完成。
