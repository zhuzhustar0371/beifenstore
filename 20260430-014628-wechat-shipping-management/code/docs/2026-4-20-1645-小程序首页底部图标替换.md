# 小程序“首页”底部图标替换记录

- 记录时间：2026-04-20 16:41:58 +08:00
- 操作人：Codex
- 目标：
  - 将小程序底部导航“首页”图标替换为基于用户提供图片的正式图标资源
- 用户提供的源文件：
  - `C:\Users\朱鑫\Downloads\首页.png`

## 背景判断

- 当前小程序使用的是 `wechat-app/app.json` 里的原生 `tabBar`
- 原生 `tabBar` 的 `iconPath` 和 `selectedIconPath` 只能引用项目内本地图片资源
- 当前项目中“首页”图标对应文件已经存在：
  - `wechat-app/images/tab-home.png`
  - `wechat-app/images/tab-home-active.png`
- 这两个文件此前是 `1x1` 的占位 PNG，无法作为正式底部栏图标使用

## 源图特征判断

- 源文件尺寸：
  - 宽 `201`
  - 高 `200`
- 可见像素区域：
  - `0,12 -> 199,175`
- 源图主体是透明背景上的房子轮廓图标
- 主体颜色为黑色，并带有透明边缘抗锯齿像素

## 本次处理方案

- 不改 `wechat-app/app.json`
  - 原因：现有 `tabBar` 文件名引用关系已经正确，只需要替换同名资源即可立即生效
- 将用户提供的原图保留到项目中，作为后续继续调整图标时的参考源
  - 新增文件：`wechat-app/images/tab-home-reference.png`
- 从原图中提取可见 alpha 区域，去掉多余透明边缘
- 将主体图形按比例缩放后居中放入 `81x81` 透明画布
- 生成首页图标两套资源：
  - `wechat-app/images/tab-home.png`
  - `wechat-app/images/tab-home-active.png`

## 视觉处理说明

- 未选中态 `tab-home.png`
  - 使用与当前 tabBar 未选中文字接近的灰色
  - 目的是降低视觉权重，符合未激活态表现
- 选中态 `tab-home-active.png`
  - 使用接近黑色的深色版本
  - 目的是保持用户提供图标的原始轮廓和识别度
- 输出格式使用透明背景 PNG
  - 原因：兼容小程序原生 `tabBar`
  - 原因：避免白底导航栏上出现额外底色块
