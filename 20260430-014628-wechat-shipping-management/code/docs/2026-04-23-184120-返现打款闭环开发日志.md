## 5. 本次代码变更

### 5.1 backend-api

- `src/main/java/com/zhixi/backend/model/CashbackRecord.java`
  - 新增 `transferDetailId`，记录微信转账明细单号。
  - 新增 `transferFailReason`，记录失败原因。

- `src/main/resources/schema.sql`
  - `cashback_records` 新增：
    - `transfer_detail_id VARCHAR(64) NULL`
    - `transfer_fail_reason VARCHAR(512) NULL`

- `src/main/java/com/zhixi/backend/config/DatabaseMigrationRunner.java`
  - 增加 `cashback_records.transfer_detail_id` 兼容迁移。
  - 增加 `cashback_records.transfer_fail_reason` 兼容迁移。

- `src/main/java/com/zhixi/backend/mapper/CashbackRecordMapper.java`
  - 所有返现查询补充 `transfer_detail_id AS transferDetailId`。
  - 所有返现查询补充 `transfer_fail_reason AS transferFailReason`。
  - 新增 `markTransferProcessing`：仅 `PENDING` 状态可原子标记为 `PROCESSING`，并写入稳定商户批次号/明细号。
  - 新增 `updateTransferResult`：统一更新状态、批次号、明细号、微信单号和失败原因。

- `src/main/java/com/zhixi/backend/service/WechatPayService.java`
  - `transferToWallet` 使用稳定单号：`ZXCB{id}` / `ZXCBD{id}`，避免重试生成多套商户单号。
  - 金额统一用 `movePointRight(2).longValueExact()` 转分，禁止精度不合法或小于等于 0 的金额。
  - 捕获微信 `ServiceException` / `HttpException`，返回可读业务错误。
  - 新增 `queryTransferDetailByOutNo`，按商户批次号/明细号同步微信转账明细。

- `src/main/java/com/zhixi/backend/service/AdminManageService.java`
  - `transferCashback` 改为返回更新后的 `CashbackRecord`。
  - 打款前先落库 `PROCESSING + outBatchNo + outDetailNo`，再发起微信商家转账；网络调用不包长事务，异常后仍保留可同步依据。
  - 微信受理成功后按批次状态映射到 `TRANSFERRED` / `FAILED` / `PROCESSING`。
  - 微信发起失败时落库 `FAILED + transferFailReason`。
  - 新增 `syncCashbackTransfer`，调用微信明细查询并同步最终状态、微信明细单号、失败原因。
  - 增加微信失败枚举的中文原因映射。

- `src/main/java/com/zhixi/backend/controller/AdminController.java`
  - `POST /api/admin/cashbacks/{cashbackId}/transfer` 返回更新后的返现记录。
  - 新增 `POST /api/admin/cashbacks/{cashbackId}/transfer/sync`。

### 5.2 zhixi-website/admin-frontend

  - 新增 `syncCashbackTransfer(cashbackId)`。

- `src/views/CashbacksPage.vue`
